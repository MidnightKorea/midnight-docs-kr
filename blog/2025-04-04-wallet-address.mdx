Midnight 생태계가 진화하고 있습니다. 중요한 변경 사항 중 하나는 새로운 기본 주소 형식인 Bech32m입니다. 오류 감지 기능이 내장된 사람이 읽을 수 있는 주소 형식입니다.

<!--truncate-->

Midnight은 이제 지갑 주소와 공개 키의 기본 형식으로 Bech32m을 사용합니다. 이 변경으로 가독성, 안전성, 메타데이터 명확성이 향상됩니다. Hex은 지원 중단되었지만, 하위 호환성을 위해 일시적으로 계속 지원됩니다. dApp을 마이그레이션하는 방법은 다음과 같습니다.

이 글에서는 이 변경이 의미하는 바, 왜 중요한지, 그리고 애플리케이션을 어떻게 적용해야 하는지 살펴봅니다.


## Why Move Away From Hex?

16진수(base-16) 인코딩은 단순성, 폭넓은 도구 지원, 간결한 형식 덕분에 블록체인 주소를 표현하는 데 오랫동안 기본으로 사용되어 왔습니다. 바이너리 데이터에 대한 직접적이고 저수준의 뷰를 제공하여, 머신이 파싱하고 개발자가 통합하기 쉽습니다. 그러나 이러한 편의성에도 불구하고, hex에는 현대 블록체인 애플리케이션에 필요한 핵심 기능이 부족합니다. 내장된 오류 감지 기능이 없어 복사-붙여넣기 실수가 발생하기 쉽고, 네트워크나 주소 유형 같은 컨텍스트 정보를 제공하지 않아 사용자 대면 시스템에서 모호성과 위험이 증가합니다.

## Introducing Bech32m

hex 인코딩의 한계를 해결하기 위해, 블록체인 생태계에서는 원래 Bitcoin SegWit을 위해 개발된 사람이 읽을 수 있는 주소 형식인 Bech32을 도입했습니다. Bech32m은 이 형식의 개선 버전으로, Taproot 같은 새로운 암호학적 구성을 지원하도록 특별히 설계되었으며, 현재 Midnight 생태계에서 사용되고 있습니다. hex과 달리 Bech32m은 시각적으로 혼동되기 쉬운 문자(예: 0 vs O)를 제외한 base-32 문자 세트를 사용하여 데이터를 인코딩하며, 오타나 잘못된 복사로 인한 오류를 감지하기 위한 강력한 체크섬을 포함합니다.

Bech32m 문자열은 세 부분으로 구성됩니다: 네트워크와 주소 유형을 식별하는 사람이 읽을 수 있는 접두사(HRP, 예: Midnight 테스트넷의 경우 `mn_shield-addr_test`), 구분자(1), 그리고 실제 데이터(예: 코인 공개 키와 암호화 공개 키)를 포함하는 인코딩된 페이로드입니다. 결과적으로 사용자가 다루기에 더 안전하고 쉬울 뿐만 아니라, 메타데이터가 풍부하여 명확성과 보안이 중요한 탈중앙화 애플리케이션에 훨씬 적합한 형식입니다.

## Where Bech32m Is Used in Midnight

Bech32m의 채택은 단일 라이브러리에 국한되지 않습니다. 전체 Midnight 지갑 생태계에 걸친 조율된 전환입니다. Wallet SDK 4.0, Wallet API 4.0, DApp Connector API v2.0, 그리고 최신 버전의 Midnight Lace Wallet 출시와 함께, Bech32m이 지갑 주소와 공개 키의 기본 인코딩이 되었습니다.
모든 핵심 컴포넌트가 기본적으로 Bech32m으로 인코딩된 필드를 노출하도록 업데이트되었으며, 레거시 hex 필드는 지원 중단으로 표시되었습니다. 이를 통해 여전히 16진수 형식에 의존하는 dApp과 인프라의 원활한 마이그레이션 경로를 보장하면서, 미래 대비성이나 안전성을 저해하지 않습니다.

실제로 `DAppConnectorWalletState` 인터페이스에서 이것이 어떻게 보이는지 확인하세요:

```
interface DAppConnectorWalletState {
  address: string; // ✅ Bech32m-encoded address

  /** @deprecated please use the `address` field instead. */
  addressLegacy: string;

  coinPublicKey: string; // ✅ Bech32m-encoded coin public key

  /** @deprecated please use the `coinPublicKey` field instead. */
  coinPublicKeyLegacy: string;

  encryptionPublicKey: string; // ✅ Bech32m-encoded encryption public key

  /** @deprecated please use the `encryptionPublicKey` field instead. */
  encryptionPublicKeyLegacy: string;
}
```

## Backward Compatibility and Migration

Bech32m이 이제 주요 형식이지만, 레거시 hex 필드는 `Legacy` 접미사를 통해 여전히 접근할 수 있습니다. 다만 이러한 필드는 공식적으로 지원 중단되었으며 향후 버전에서 제거될 예정입니다. dApp이나 도구가 여전히 hex 값을 사용한다면, SDK에서 변환을 돕는 유틸리티 패키지를 제공합니다:

```
import { decodeBech32mAddress } from '@midnight-ntwrk/wallet-sdk-address-format';

const hexAddress = decodeBech32mAddress(address);
```

이를 통해 레거시 시스템과 계속 작업하면서 더 안전하고 표현력이 풍부한 Bech32m 형식으로 점진적으로 마이그레이션할 수 있습니다.

## What You Should Change

DApp Connector API나 Wallet SDK를 통해 지갑 상태를 사용하고 있다면, 다음 사항을 확인하세요:

✅ `addressLegacy → address`로 전환

✅ `coinPublicKeyLegacy → coinPublicKey`로 전환

✅ `encryptionPublicKeyLegacy → encryptionPublicKey`로 전환

✅ 필요한 경우 `@midnight-ntwrk/wallet-sdk-address-format`을 사용하여 변환

예시:

```
import { decodeBech32mAddress } from '@midnight-ntwrk/wallet-sdk-address-format';

const hex = decodeBech32mAddress("mtst1q0yz7f64vu...");
```

## TL;DR: Bech32m Is Now the Default

✅ Bech32m이 모든 지갑 관련 패키지에서 주소/공개 키의 기본 형식입니다

❌ Hex 형식 필드(addressLegacy, coinPublicKeyLegacy 등)는 지원 중단되었습니다

🛠️ 마이그레이션은 간단합니다. 이미 제공된 새 필드와 타입을 사용하면 됩니다

🔁 레거시 hex 값은 하위 호환성을 위해 일시적으로 계속 사용할 수 있습니다

📦 dApp을 개발 중이라면, Wallet SDK 4.0.0과 이전 버전을 모두 지원하는 `midnight-js@1.0.0`을 사용하세요. Bech32m 주소와의 호환성도 지원합니다.
