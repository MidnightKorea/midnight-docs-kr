# How to Connect a dApp to the Midnight Network Using the Lace Wallet

> 이 블로그 포스트는 MidnightJS 버전 1.0.0을 기준으로 합니다. 여기서 설명하는 단계는 향후 버전에서 변경될 수 있습니다.

데이터 보호에 중점을 둔 네트워크인 Midnight 블록체인이 주목을 받으면서, 탈중앙화 애플리케이션(dApp)을 개발하는 개발자에게는 사용자를 네트워크에 안전하고 편리하게 연결하여 상호작용할 수 있는 방법이 필요합니다.

Midnight 팀이 개발한 Cardano용 Lace 지갑의 특별 에디션인 [Midnight용 Lace 지갑](https://docs.midnight.network/relnotes/lace)은 브라우저 window에 안전한 인터페이스를 직접 주입하여 이 과정을 매끄럽게 만들어 줍니다.

> 참고: Midnight과 상호작용하기 위해 전용 지갑을 사용하는 것은 임시 방편입니다. 향후 Midnight이 Lace 지갑에 직접 통합될 예정입니다.

이 글에서는 Lace 지갑을 감지하고, 연결하고, 상호작용하는 데 필요한 정확한 단계를 살펴보며, dApp에서 Midnight 네트워크를 활용할 수 있도록 안내합니다.

## 🧭 Step 1: Detect the Lace Wallet
Lace 지갑은 [DappConnectorApi](https://docs.midnight.network/develop/reference/midnight-api/dapp-connector/)의 인스턴스를 전역 `window` 객체의 `window.midnight.mnLace`에 주입합니다. Dapp Connector API를 통해 dApp과 지갑 간의 상호작용을 구축하고, 지갑에서 정보(예: 사용자 주소)를 읽는 데 사용할 수 있는 다양한 속성과 메서드에 접근할 수 있습니다.

가장 먼저 해야 할 일은 해당 객체의 존재 여부를 확인하는 것입니다:
```ts
if (window.midnight && window.midnight.mnLace) {
  const mnLace = window.midnight.mnLace;
  // Proceed with next steps
}
```
이 객체가 존재하지 않으면, 공식 다운로드 페이지 링크를 통해 사용자에게 지갑 설치를 안내할 수 있습니다: https://docs.midnight.network/relnotes/lace

## 🔒 Step 2: Check if the Wallet is Authorized
지갑과 상호작용하기 전에, 사용자가 이미 dApp에 지갑 접근 권한을 부여했는지 확인해야 합니다:
```ts
const isEnabled = await mnLace.isEnabled();
```
isEnabled가 `true`이면, dApp이 이미 인증되었으며 지갑 기능에 접근할 수 있습니다. 그렇지 않으면, 명시적으로 접근을 요청해야 합니다(다음 단계).

## ✅ Step 3: Request Access (Enable)
사용자가 아직 dApp을 인증하지 않았다면, 다음을 호출하여 연결을 요청하세요:
```ts
const walletApi = await mnLace.enable();
```
이렇게 하면 Lace 지갑에서 사용자에게 연결 승인을 요청하는 UI 팝업이 표시됩니다. 승인되면, 전체 지갑 API 인터페이스에 접근할 수 있습니다.
> 참고: 앱 로딩 시 자동으로 팝업을 띄우는 대신, 사용자가 인터페이스와 상호작용하여 지갑을 승인하도록 하는 것이 일반적으로 권장됩니다.

## 🧾 Step 4: Fetch the User's Address
연결이 완료되면, walletApi에서 제공하는 `state()` 메서드를 사용하여 지갑에서 사용자 정보를 조회할 수 있습니다:
```ts
const state = await walletApi.state();
const userAddress = state.address;
```

이를 통해 사용자의 Midnight 네트워크 주소를 얻을 수 있으며, UI에 표시하는 등 다양한 용도로 활용할 수 있습니다.

## 🔧 Step 5: Get Service Configuration (Optional)
지갑은 [서비스 엔드포인트](https://docs.midnight.network/develop/reference/midnight-api/dapp-connector/interfaces/ServiceUriConfig)와 같은 추가 구성 메타데이터도 제공합니다. 다음과 같이 가져올 수 있습니다:
```ts
const config = await mnLace.serviceUriConfig();
```
지갑이 현재 연결된 노드, 인덱서, proof server의 URL이 포함된 객체를 반환합니다.

dApp이 현재 환경에 따라 달라지는 특정 Midnight 서비스나 API와 통합되는 경우 유용합니다. 예를 들어 트랜잭션을 증명하기 위해 proof server의 주소가 필요한 경우에 활용할 수 있습니다.

## 💡 Example Flow
연결 로직이 일반 JavaScript에서 어떻게 보일 수 있는지 간략한 흐름을 보여드립니다:
```ts
async function connectToMidnight() {
    if (window.midnight && window.midnight.mnLace) {
        const mnLace = window.midnight.mnLace;
        const walletApi = await mnLace.isEnabled();

        const state = await walletApi.state();
        console.log("User address:", state.address);

        const config = await mnLace.serviceUriConfig();
        console.log("Service config:", config);
    } else {
        alert("Please install the Lace wallet to connect to Midnight.");
    }
}
```

위 코드 스니펫이 Lace 지갑을 통해 Midnight에 연결하기 위해 수행하는 작업의 간략한 체크리스트입니다:
1. `window.midnight.mnLace` 객체를 감지합니다.
2. `isEnabled()`로 인증을 확인합니다.
3. 아직 인증되지 않은 경우 `enable()`을 사용하여 접근을 요청합니다.
4. state()을 사용하여 Midnight 주소 같은 사용자 정보를 가져옵니다.
5. 고급 용도를 위해 선택적으로 서비스 구성을 가져옵니다.

이러한 간단하지만 안전한 단계를 통해, dApp은 Lace 지갑을 사용하여 Midnight 블록체인에 완전히 연결되며, 사용자에게 브라우저에서 바로 매끄럽고 프라이버시를 보장하는 경험을 제공합니다.

## 🧠 Conclusion
Midnight 네트워크를 위해 dApp에 Lace 지갑을 통합하는 것은 보안과 사용자 경험 모두를 우선시하는 간단한 과정입니다.  
지갑의 존재 감지, 인증 확인, 접근 요청, 사용자 데이터 조회 등 설명된 단계를 따르면, 프라이버시와 안정성을 유지하면서 사용자를 Midnight 블록체인에 매끄럽게 연결할 수 있습니다.  
전용 Lace 지갑 에디션에 대한 임시 의존은 현재의 호환성을 보장하지만, 개발자는 이 과정을 더 간소화할 수 있는 향후 업데이트에 대해 인지하고 있어야 합니다.  
Midnight이 발전함에 따라, 이 가이드라인을 준수하면 dApp이 네트워크의 표준에 부합하여 사용자가 애플리케이션과 안전하게 상호작용할 수 있게 됩니다. 구현이 최신 상태를 유지하도록 최신 API 변경 사항과 모범 사례에 대해 항상 [공식 문서](https://docs.midnight.network/)를 참조하세요.
