# Learning Web3 from the Ground Up: Smart Contracts and the Compact Language

Web3 세계를 탐험하면서 Midnight 같은 프라이버시 우선 플랫폼을 가능하게 하는 핵심 구성 요소를 파고들고 있습니다(이전 글은 Midnight Dev Diaries에서 확인하세요!). 이번 주에는 밀접하게 연관된 두 가지 기초 요소에 집중했습니다: Midnight의 smart contract와 이를 구동하는 Compact 언어입니다.

Smart contract는 blockchain에서 새로운 개념이 아니지만, Midnight의 접근 방식은 독특합니다. 코드를 온체인에서 직접 실행하는 대신, Midnight은 smart contract를 사용하여 오프체인에서 zero-knowledge proof를 통해 적용되는 규칙을 정의합니다. 이 시스템의 중심에 있는 것이 바로 [Compact](https://docs.midnight.network/develop/reference/compact/compact-reference)입니다. 프라이버시를 보존하는 contract를 안전하고 실용적으로 작성할 수 있도록 설계된 전용 언어입니다.

이것이 실제로 어떻게 작동하는지 더 잘 이해하기 위해 [Midnight Developer Academy의 Module 2](https://docs.midnight.network/academy/module-2)도 진행했습니다. 이 모듈에서는 contract 구조, circuit 유형, contract와 이를 실행하는 외부 로직 간의 관계 등 핵심 개념을 다룹니다.

이번 글에서는 Midnight의 smart contract가 어떤 점에서 다른지, Compact의 구조는 어떠한지, 그리고 이 둘이 어떻게 협력하여 민감한 데이터를 노출하지 않으면서 증명 가능한 로직을 구현하는지 설명하겠습니다.

## What Is a Smart Contract on Midnight?

큰 그림에서 보면, smart contract는 blockchain에 저장된 자율 프로그램입니다. 상호작용 규칙을 정의하고, 해당 규칙이 충족되었는지 확인하며, 결과를 기록합니다. 이 모든 것이 신뢰할 수 있는 제3자 없이 이루어집니다.

하지만 Midnight은 일반적인 blockchain이 아닙니다. 컴플라이언스를 고려한 프라이버시 우선 플랫폼이므로, smart contract가 단순히 로직을 실행하는 것 이상을 해야 합니다. 개인 데이터를 보호하고, selective disclosure를 가능하게 하며, 적대적인 환경에서도 보안과 신뢰를 유지해야 합니다.

여기서 Compact이 등장합니다. Midnight에서 smart contract를 작성하기 위해 명시적으로 설계된 도메인 특화 언어입니다.

기존의 많은 smart contract처럼 로직을 온체인에서 직접 실행하는 대신, Compact contract는 증명할 로직을 정의합니다. 실행 환경 자체가 아닙니다. Contract는 준수해야 할 규칙을 인코딩하고, 이 규칙이 zero-knowledge proof의 기반으로 사용됩니다. 실제 실행, 즉 transaction 처리, 자격 증명 생성, 투표 제출 등은 외부에서 이루어집니다. 보통 DApp, 백엔드 서비스, 또는 API에서 처리합니다.

해당 외부 시스템이 필요한 로직을 실행하고, 암호학적 proof를 생성하여 contract의 규칙을 따랐음을 **증명**합니다. Blockchain은 세부 사항을 알 필요가 없으며, proof가 유효한지만 검증하면 됩니다. 이러한 분리 구조가 Midnight의 모델을 확장 가능하고 프라이버시를 보존할 수 있게 만듭니다.

즉, contract가 규칙을 설정하고, 앱이 로직을 실행하며, blockchain은 민감한 입력값을 보지 않고 결과를 검증합니다.

## What Is Compact?

Compact은 Midnight 네트워크의 고유한 목표에 맞춰 설계된 정적 타입의 프라이버시 중심 smart contract 언어입니다. JavaScript나 TypeScript를 다뤄본 적이 있다면 Compact이 상당히 친숙하게 느껴질 것입니다. 정적 타입의 circuit를 사용해 오프체인 연산에 매핑되는 zero-knowledge proof 로직을 정의하는 도메인 특화 언어(DSL)입니다. 하지만 간결한 문법에 속으면 안 됩니다. Compact은 내부적으로 zero-knowledge proof(ZKP)를 사용한 프라이버시 보존 연산을 위해 설계되었습니다. (Compact에 대한 더 기술적인 심화 내용은 Kevin Millikin의 Compact Deep Dive [여기](https://docs.midnight.network/blog/compact)에서 확인하세요!)

Compact contract는 세 가지 핵심 구성요소로 이루어집니다:

- **Ledger**: Contract의 공개적인 온체인 상태입니다.
- **Circuit**: 데이터를 조작하고, proof를 생성하며, ledger와 상호작용하는 오프체인 로직입니다.
- **Witness**: 비공개로 보유된 데이터로, contract를 실행하는 주체만 알고 있습니다.

Compact은 [explicit disclosure](https://docs.midnight.network/develop/reference/compact/explicit-disclosure#conclusion)도 지원합니다. 개발자가 어떤 데이터를 누구에게 공개할지 정확히 정의할 수 있습니다. 이를 통해 온체인에서 무엇이 보이고 무엇이 비공개로 유지되는지 세밀하게 제어할 수 있으며, 비공개 자산 전송, 연령 인증 접근, 기밀 투표 등의 사용 사례에 필수적입니다.

## Circuits and Privacy

Midnight Developer Academy Module 2에서 가장 중요한 핵심 사항 중 하나는 circuit의 작동 방식입니다. Compact에서 circuit은 사용자나 시스템이 호출하여 작업을 트리거할 수 있는 함수입니다. 하지만 Web2 API나 EVM contract 함수와 달리, 이 circuit은 blockchain에서 직접 로직을 실행하지 않습니다. 대신 오프체인에서 실행되고 로직이 준수되었다는 zero-knowledge proof를 생성합니다.

실제로 이는 다음을 의미합니다:

- Contract 로직은 공개입니다.
- 실행 중 사용되는 데이터는 비공개입니다.
- 로직의 결과(예: "규칙이 준수되었음")는 proof를 사용하여 공개적으로 검증 가능합니다.

이것이 Midnight 모델의 강점입니다. 사용자는 결과를 유발한 정확한 데이터를 공개하지 않고도 규칙을 준수했음을 증명할 수 있습니다. "투표 자격이 있음을 증명했다"고 생각해 보세요. 자신이 누구인지, 무엇에 투표했는지는 공개하지 않으면서 말입니다.

## From Theory to Practice

기존의 smart contract 플랫폼에서는 기본적으로 모든 것이 공개됩니다: 데이터, 로직, 입력, 출력. 투명성에는 유용하지만, 건강 데이터, 금융 기록, 신원 확인 같은 민감한 사용 사례에는 항상 실용적이지 않습니다.

Midnight은 이 기본값을 뒤집습니다. 프라이버시가 내장되어 있으며, 개발자는 필요한 경우에만 선택적으로 공개합니다.

Midnight에서 Compact으로 구축할 수 있는 사례는 다음과 같습니다:

- 결과만 공개되는 투표 시스템
- 신원을 드러내지 않고 사용자 속성을 확인하는 접근 제어 시스템
- 금액이나 발신자를 숨기면서도 공개 검증이 가능한 토큰 전송 앱

## Conclusion

Smart contract는 탈중앙화 애플리케이션의 기반이지만, Midnight에서는 프라이버시 우선의 미래를 위해 설계되었습니다. 모든 로직을 온체인에서 실행하는 대신, Midnight은 contract의 로직과 실행을 분리합니다. Compact을 통해 개발자는 zero-knowledge proof 시스템의 규칙을 정의하여, 민감한 데이터를 노출하지 않으면서 사실을 증명할 수 있는 애플리케이션을 구현할 수 있습니다. 이는 안전하고 확장 가능하며 사용자를 보호하는 Web3 설계를 향한 실질적인 전환입니다.

Midnight에서 이것은 이론적 개선에 그치지 않습니다. 플랫폼 운영의 핵심입니다. Compact smart contract를 통해 개발자는 입력과 실행을 오프체인에 유지하면서 검증 가능하고 감사 가능한 로직을 작성할 수 있습니다. ZKP와 shielded transaction과 결합하면, 프라이버시나 탈중앙화를 타협하지 않으면서 실제 사용 사례를 지원하는 인프라가 만들어집니다.

Web3에서 smart contract를 구축하는 방법을 배우고 있거나 이 퍼즐 조각들이 어떻게 맞춰지는지 궁금하다면, [Midnight Developer Academy의 Module 2](https://docs.midnight.network/academy/module-2)와 [Compact Language Reference](https://docs.midnight.network/develop/reference/compact/compact-reference)를 살펴보시기 바랍니다. 이 자료들은 개념을 단계별로 설명하고 Midnight의 아키텍처가 어떻게 고급 암호학을 실용적인 개발 도구로 전환하는지 보여줍니다.

다음 주에도 탈중앙화 세계에서 프라이버시 우선 시스템을 구성하는 요소들을 계속 풀어나가겠습니다.
