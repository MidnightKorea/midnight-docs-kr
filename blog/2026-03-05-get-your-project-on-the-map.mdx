# Get your project on the map

*여러분의 작업이 Midnight 생태계의 일부로 가시적이고, 집계되고, 인정받도록 하기 위한 세 가지 빠른 업데이트입니다.*

## Why this matters

Midnight은 메인넷에 다가가고 있습니다. 이미 수백 명의 활발한 개발자가 Compact으로 개발하고, Preprod에 배포하며, 실제 애플리케이션을 출시하고 있습니다. 하지만 **귀속되지 않은 활동은 집계되지 않으며**, 이러한 비가시성은 여러분이 구축하는 데 기여하는 생태계에 실질적인 영향을 미칩니다.

개발 활동의 명확한 공개 기록을 확립하면 더 넓은 블록체인 업계가 Midnight Network의 성장을 인지하게 됩니다. 정확한 귀속은 생태계의 기능적 필수 사항이며, 기술적 기여를 문서화하기 위한 표준화된 프레임워크를 확립합니다.

## The attribution process

[**Electric Capital**](https://www.developerreport.com/)은 블록체인에서 가장 널리 인용되는 개발자 리포트를 발행합니다. 투자자, 프로토콜, 미디어는 이를 통해 어떤 생태계가 성장하고 있는지, 어떤 것이 정체되어 있는지, 어디에 주목해야 하는지 파악합니다. 그들의 방법론은 공개되어 있습니다. 모든 주요 블록체인 생태계의 GitHub 커밋, 기여자 수, 저장소 메타데이터를 추적합니다. 저장소에 올바른 메타데이터가 없으면, 작업이 아무리 활발하고 고품질이더라도 Midnight과 연결되지 않습니다.

Midnight은 메인넷 출시 전에 처음으로 Electric Capital에 생태계 맵을 제출합니다. 이것은 생태계가 첫날부터 어디에 있는지 신뢰할 수 있는 공개 기록을 확립할 **일회성 기회**입니다. 메인넷 출시 전후 몇 주의 개발자 수는 새로운 생태계의 순위와 보도에 영향을 미치는 롤링 윈도우에 포착됩니다. 해당 윈도우에서 강력한 수치로 나타나는 생태계는 인정받습니다. 그렇지 못한 생태계는 수개월 동안 수정하기 어려운 적자에서 시작합니다.

## Compounding developer results

아래 변경 사항은 저장소당 5분이 채 걸리지 않습니다. 작지만 영구적이고 중요한 변경입니다. 코드 작성 외에도, 이러한 메타데이터 표준을 구현하는 것은 생태계 성장을 위한 영향력 높은 행동입니다. 이를 완료하는 모든 저장소는 Midnight 위에서 개발한 것으로 인정받는 개발자가 됩니다.

| TL;DR: Midnight의 개발자 생태계는 실재하며 성장하고 있습니다. 이 세 단계를 통해 업계가 주목하는 도구와 리포트에서 그 성장이 가시적으로 드러나도록 합니다. |
| :---- |

## Step 1: Add GitHub topics

GitHub 토픽은 Electric Capital(및 기타 생태계 추적 도구)이 어떤 저장소가 어떤 생태계에 속하는지 프로그래밍 방식으로 식별하는 수단입니다. 토픽이 없으면 활발히 관리되는 저장소도 도구에서 보이지 않습니다.

저장소에 토픽을 추가하려면:

1. GitHub에서 저장소로 이동하세요

2. 오른쪽 사이드바에서 About 섹션을 찾으세요

3. 옆에 있는 기어 아이콘을 클릭하세요

4. Topics 필드에 아래 표의 관련 토픽을 추가하세요

5. Save changes를 클릭하세요

| Topic | When to use |
| :---- | :---- |
| **`midnightntwrk`** | 모든 Midnight 생태계 프로젝트에 필수 |
| **`compact`** | 프로젝트가 Compact 언어를 사용하는 경우에만 추가 |


:::warning

사용하지 마세요: `midnight`, `midnight-network`, `midnight-compact`, `midnight-ecosystem`. 이러한 변형은 추적되지 않으며 프로젝트를 생태계와 연결하지 않습니다.

:::

## Step 2: Add one attribution sentence to your README

Electric Capital의 도구는 저장소가 생태계와 관련 있다는 것을 확인하기 위해 README 파일을 스캔합니다. README 상단 근처에 한 문장이면 충분합니다. 아래 표의 **정확한 문구**를 사용하세요. 문구를 커스터마이즈하면 자동화 시스템이 인식하지 못할 수 있습니다.

| Your project type | Attribution sentence |
| :---- | :---- |
| Midnight 위에서 직접 실행되는 dApp, 컨트랙트 또는 도구 | *"This project is built on the Midnight Network."* |
| SDK, 인프라, 지갑 또는 서비스 | *"This project integrates with the Midnight Network."* |
| 개발자 도구, 프레임워크 또는 라이브러리 | *"This project extends the Midnight Network with additional developer tooling."* |

## Step 3: Open a PR to the Awesome dApps list

Midnight [Awesome dApps list](http://github.com/midnightntwrk/midnight-awesome-dapps)는 커뮤니티가 관리하는 생태계 프로젝트 디렉터리입니다. PR을 열면 해당 기록에 프로젝트가 추가되어 다른 개발자, 파트너, 생태계를 평가하는 팀이 발견할 수 있게 됩니다.

프로젝트를 제출하세요: [github.com/midnightntwrk/midnight-awesome-dapps](https://github.com/midnightntwrk/midnight-awesome-dapps).

## Three steps, once per repo, permanent record

이러한 변경은 코드의 동작 방식에 영향을 미치지 않습니다. 작업이 어떻게 보이는지에 영향을 미칩니다. Midnight의 첫 공개 개발자 수는 앞으로 수년간 네트워크를 참조하는 모든 업계 리포트, 투자자 브리핑, 생태계 비교에 사용됩니다. 여러분의 저장소가 그 수에 포함되는 것이 중요합니다.
