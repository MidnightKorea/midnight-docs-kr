작동하는 게시판 DApp의 TypeScript 코드를 살펴봅니다. 컨트랙트 상호작용 로직과 비공개 상태 관리를 포함한 전체 코드베이스를 다룹니다.


## Prerequisites

컨트랙트는 `contract/src` 하위 디렉토리에 있어야 합니다. 컨트랙트를 컴파일하면 `managed` 하위 디렉토리에 TypeScript API와 관련 파일이 생성됩니다.

:::note
Compact 컴파일러는 Compact 컨트랙트 코드에서 TypeScript API를 생성합니다. 컴파일 과정에서 컨트랙트 상호작용을 위한 타입 안전 인터페이스가 만들어집니다.
:::

TypeScript 코드는 세 개의 디렉토리에 걸쳐 있습니다:
- `contract/src` - 컨트랙트 정의 및 witness 함수
- `api/src` - 애플리케이션 추상화 계층
- `bboard-cli/src` - 메인 애플리케이션 실행 루프

핵심 파일:
- `contract/src/witnesses.ts` - 비공개 상태 및 witness 함수 정의
- `api/src/index.ts` - 애플리케이션 인터페이스 구현

## Define the private state

블록체인은 모든 사용자에게 보이는 공개 컨트랙트 상태를 저장합니다. 비공개 상태는 각 DApp 인스턴스에 로컬로 유지되며 사용자마다 다릅니다. 컨트랙트는 비공개 상태 접근 함수의 타입만 선언하고, 함수 구현이나 비공개 상태 구조는 지정하지 않습니다. 생성된 컨트랙트 API는 비공개 상태 타입으로 특정 컴포넌트를 매개변수화합니다.

비공개 상태용 인터페이스나 타입 별칭을 정의하는 것이 모범 사례입니다.

게시판의 비공개 상태에는 애플리케이션 수명 주기 동안 변하지 않는 사용자의 비밀 키가 포함됩니다. `localSecretKey()` witness가 이 값을 가져옵니다.

게시판 컨트랙트는 비밀 키를 바이트 배열로 선언하며, 이는 TypeScript의 `Uint8Array`에 해당합니다. `secretKey` 속성을 가진 `BBoardPrivateState`를 정의합니다:

```typescript
// Compact 컴파일러가 생성한 이 타입 정의는 타입 안전 비공개 상태 처리를 가능하게 합니다
export type BBoardPrivateState = {
  secretKey: Uint8Array;
};
```

다음 헬퍼 함수는 `BBoardPrivateState` 객체를 생성합니다:

```typescript
export function createBBoardPrivateState(secretKey: Uint8Array): BBoardPrivateState {
  return { secretKey };
}
```

## Initialize the private oracle

영지식 증명 시스템은 비공개 상태에 접근하는 컴포넌트를 *오라클*이라고 부릅니다. 이 용어는 Midnight API와 문서 전반에서 사용됩니다.

`witnesses` 객체는 선언된 각 witness 함수에 대한 속성 또는 메서드가 필요합니다. `bboard.compact` contract는 하나의 witness 함수를 선언합니다: `localSecretKey`.

### Function structure

함수 시그니처에는 다음이 포함됩니다:
- **매개변수**: `WitnessContext` - ledger 상태, 비공개 상태 및 contract 주소를 포함
- **반환값**: 다음을 포함하는 튜플:
  - 새 비공개 오라클 상태
  - witness 함수의 선언된 반환 타입과 일치하는 값

`WitnessContext<L, PS>` 타입은 다음을 포함합니다:
- `ledger: L`
- `privateState: PS`
- `contractAddress: string`

이 구현은 다음을 사용합니다:
- Compact 컴파일러가 생성한 API의 `Ledger` 타입
- Exercise 1의 `BBoardPrivateState`

### Implementation

`localSecretKey` 함수는 변경되지 않은 비공개 상태(상태를 수정하지 않으므로)와 컨트랙트 해시 생성 및 검증에 사용할 비밀 키 값을 반환합니다:

```typescript
// 이 witness 함수는 Compact 컴파일러가 생성한 contract API와 통합됩니다
export const witnesses = createBBoardWitnesses<BBoardPrivateState>({
  localSecretKey: ({ privateState }) => {
    return [privateState, privateState.secretKey];
  },
});
```

:::important
비공개 상태를 직접 변경하지 마세요. witness 함수에서 새 상태 값을 반환하세요.
:::

:::caution
비공개 상태에 전역 변수를 사용하지 마세요. 항상 witness 함수에 전달된 값을 사용하세요.
:::

## Invoke the post circuit

`api/src/index.ts` 파일은 배포된 게시판 컨트랙트의 애플리케이션 인터페이스인 `BBoardAPI`를 정의합니다.

### Post method implementation

`post` 메서드는 게시판에 새 메시지를 제출합니다. 이 비동기 메서드는 `DeployedBBoardContract`로 초기화된 `BBoardAPI` 클래스에 속합니다.

`DeployedBBoardContract`는 [`FoundContract`](../../reference/midnight-api/midnight-js/@midnight-ntwrk/midnight-js-contracts/type-aliases/FoundContract)의 별칭으로, 배포된 스마트 컨트랙트에 대한 Midnight.js의 추상화입니다. 애플리케이션에 비공개 배포 메타데이터가 필요 없으므로 `DeployedContract` 대신 `FoundContract`를 사용합니다.

`FoundContract`는 각 컨트랙트 circuit에 대한 함수를 제공하는 `callTx`를 노출하며, 이 함수들이 트랜잭션을 생성하고 제출합니다.

게시판의 `post` 메서드는 circuit를 호출합니다:

```typescript
// Compact 컴파일러가 생성한 API는 타입 안전 circuit 호출을 제공합니다
await this.deployedContract.callTx.post([message]);
```

비어 있지 않은 게시판에 게시하면 트랜잭션이 실패하고 예외가 발생합니다. 현재 구현에서 예외는 메인 `run` 함수로 전파되어 DApp이 종료됩니다. 테스트 후 더 나은 오류 처리를 원한다면 try/catch 블록 추가를 검토하세요.

## Invoke the take-down circuit

`takeDown` 함수는 `api/src/index.ts`에서 `post` 메서드 뒤에 있습니다.

이 메서드는 배포된 컨트랙트의 `callTx` 속성을 통해 `takeDown` circuit를 호출합니다:

```typescript
// Compact 컴파일러는 circuit 매개변수에 대한 타입 안전성을 보장합니다
await this.deployedContract.callTx.takeDown([]);
```

## Deploy a new bulletin board contract

컨트랙트 circuit 호출과 트랜잭션 제출에는 최소한의 코드만 필요합니다. 컨트랙트 배포는 설정이 좀 더 필요하지만 여전히 간결합니다.

### Deployment function

`api/src/index.ts`의 `deploy` 함수는 필요한 모든 인자를 포함한 `deployContract` 호출을 포함합니다.

`deployContract`는 두 개의 인자가 필요합니다:

- **MidnightProviders 객체**: 필요한 모든 provider 구현을 포함
- **DeployContractOptions 객체**: 배포 구성 매개변수를 포함

게시판 컨트랙트의 경우 `DeployContractOptions`에 다음이 필요합니다:
- `privateStateKey`: `PrivateStateProvider`에 비공개 상태를 저장할 키
- `contract`: Compact 컴파일러가 생성한, 실행 가능한 JavaScript를 포함하는 `Contract` 객체
- `initialPrivateState`: `privateStateKey`로 저장되는 타입과 일치하는 초기 비공개 상태

### Configuration

#### Private state key

```typescript
privateStateKey: 'bboard-private-state'
```
구성된 provider에서 사용자 비공개 상태를 검색하기 위한 고유 식별자입니다. 기존 상태가 없으면 시스템이 이 키 아래에 새 상태를 생성하고 저장합니다.

#### Contract instance

```typescript
contract: bboardContractInstance
```

다음과 같이 정의됩니다:
```typescript
// Compact 컴파일러는 witness 함수 통합이 포함된 Contract 클래스를 생성합니다
export const bboardContractInstance = new Contract(witnesses);
```

앞서 정의한 witness 함수로 Compact 컴파일러가 생성한 `Contract` 인스턴스를 만듭니다.

#### Initial private state

```typescript
initialPrivateState: await getPrivateState(providers.privateStateProvider)
```

`getPrivateState` 메서드는 기존 비공개 상태를 가져오거나 32바이트 랜덤 값으로 새 상태를 생성하여, 항상 유효한 비공개 상태에 접근할 수 있도록 합니다.

### Complete deployment call

```typescript
// Compact 컴파일러가 생성한 Contract 클래스는 타입 안전 배포를 가능하게 합니다
const deployedContract = await deployContract(providers, {
  privateStateKey: 'bboard-private-state',
  contract: bboardContractInstance,
  initialPrivateState: await getPrivateState(providers.privateStateProvider),
});
```

이 코드만으로 Midnight 블록체인에 컨트랙트를 배포할 수 있습니다.

### Join existing contracts

`join` 함수(`deploy` 아래)에는 기존 컨트랙트에 참여하기 위한 유사한 코드가 있습니다. 두 구현을 비교하여 차이점을 확인해 보세요.

## Compile and run the DApp

`example-bboard` 디렉토리(`contract`, `api`, `bboard-cli` 포함)로 이동하여 실행합니다:

```shell
npx turbo build
```

:::note
빌드 프로세스는 애플리케이션 전체의 타입 안전성을 보장하기 위해 `managed` 디렉토리의 Compact 컴파일러 출력을 사용합니다.
:::

### Troubleshooting: ERR_UNSUPPORTED_DIR_IMPORT

이 Node.js 오류는 `.zshrc`, `.bashrc`를 수정하거나 NVM으로 Node 버전을 변경한 뒤 환경 캐시 문제로 발생합니다.

**해결 방법:**
1. 새 터미널 창을 엽니다(`source ~/.zshrc`에 의존하지 마세요)
2. Node 버전을 확인합니다: `nvm use 18`
3. 캐시된 모듈을 삭제합니다:
   ```bash
   rm -rf node_modules/.cache
   ```

이 환경 관련 문제는 오래된 Node.js 설정에서 특정 ESM 스타일 임포트에 영향을 줍니다.
