# counter 컨트랙트

이 파트에서는 먼저 Compact 컨트랙트를, 이어서 counter DApp의 TypeScript 코드를 살펴봅니다.

counter DApp의 기반이 되는 컨트랙트는 Counter DApp 예제 디렉토리의 다음 경로에 있습니다:

```
example-counter/contract/
```

counter 컨트랙트 자체는 매우 간단합니다. `counter.compact`의 전체 내용은 다음과 같습니다:

import CodeFromGithub from '@site/src/components/CodeFromGithub';

<CodeFromGithub
  url="https://github.com/midnightntwrk/example-counter/blob/main/contract/src/counter.compact"
  language="compact"
/>



이 Compact 코드를 이해하려면 중간의 `ledger` 필드 선언부터 시작합니다. 이 선언은 Midnight 블록체인에서 볼 수 있는 컨트랙트의 공개 상태에 `round` 필드가 포함됨을 나타냅니다. Midnight ledger는 [상태 타입](../../reference/compact/compact-reference#ledger-state-types) 중 하나로 `Counter`를 지원하며, 이 필드가 해당 타입으로 선언되어 있습니다.

ledger 필드 선언 외에도, 컨트랙트는 비공개 상태에 대한 인터페이스를 제공하는 함수를 선언할 수 있으며 각각 `witness` 키워드로 표시됩니다. counter 컨트랙트는 이런 숨겨진 오프체인 상태를 사용하지 않으므로 witness 선언이 없습니다.

이 컨트랙트는 사용자가 호출할 수 있는 하나의 공개 연산(_circuit_)인 `increment`를 제공합니다. `increment` 연산은 `round` 카운터 값을 `1`만큼 증가시킵니다.

파일 상단에 두 줄의 코드가 더 있습니다. 첫 번째 줄은 Compact 언어 버전 제약 조건을 지정하는 `pragma`입니다. Compact 컴파일러가 지정된 언어 버전을 지원하지 않으면 컴파일 오류가 발생합니다. 두 번째는 Compact 표준 라이브러리(`CompactStandardLibrary`)를 사용할 수 있게 하는 `import`입니다. 표준 라이브러리에 ledger 타입 `Counter`가 포함되어 있으므로 임포트가 필요합니다.

더 복잡한 컨트랙트에서는 ledger용 열거형과 구조체 타입, 여러 circuit(다른 circuit에서만 호출되는 비공개 circuit 포함), 비공개 오프체인 상태를 다루는 함수 등을 선언할 수 있습니다. 이 튜토리얼의 다음 주요 섹션인 게시판 예제에서 Midnight 컨트랙트에 대해 훨씬 더 자세히 다룹니다.
