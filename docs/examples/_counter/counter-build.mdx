# Build the counter DApp

코드를 간결하게 유지하기 위해, counter DApp은 웹 인터페이스 대신 텍스트 기반 사용자 인터페이스를 사용합니다.

`example-counter` 저장소 루트에는 각각 별도로 빌드할 수 있는 두 개의 하위 디렉토리가 있습니다.
- `contract` - counter 컨트랙트의 Compact 소스 코드와 관련 TypeScript
- `counter-cli` - DApp의 커맨드 라인 인터페이스이며, `contract` 코드에 의존


## Compile the code

1. `contract` 폴더에서 Compact 컨트랙트를 컴파일하고 빌드합니다:

```shell
npm run compact && npm run build
```

`compact` 스크립트는 `package.json`에 설정된 경로로 `compact compile`을 호출합니다. `build` 스크립트는 TypeScript 래퍼 코드를 컴파일합니다.

2. `counter-cli` 폴더로 이동하여 애플리케이션을 실행합니다:

```shell
npm run start-testnet-remote
```

DApp이 시작되고 원격 Midnight testnet 노드에 연결됩니다.

:::note
npm 스크립트는 내부적으로 `compact compile` 명령을 사용합니다. 더 이상 사용되지 않는 `compactc` 실행 파일은 사용하지 마세요. 항상 `compact compile` 또는 제공된 npm 스크립트를 사용하세요.
:::

다음과 같은 출력이 표시됩니다:

```
Compact version: 0.25.0
Compiling 1 circuits:
  circuit "increment" (k=10, rows=29)
Overall progress [====================] 1/1
```

:::note
`--skip-zk` 옵션으로 ZK 키 생성을 건너뛰었기 때문에 컴파일이 매우 빠르게 완료됩니다. 출력 파일은 `contract/src/managed/counter` 디렉토리에 저장됩니다.
:::

## Build the TypeScript source files

1. `contract` 폴더에서 다음 명령을 실행합니다:

```shell
npm run build
```

`contract/dist` 디렉토리가 생성됩니다.

2. 다음 명령으로 유닛 테스트를 시작합니다:

```shell
npm run test
```

3. `counter-cli` 폴더에서 다음 명령을 실행합니다:

```shell
npm run build
```
