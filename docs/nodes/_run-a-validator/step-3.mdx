# Register SPO as a candidate in the block producer committee

## Prerequisites

- Cardano SPO를 운영 중이며 다음 Cardano 키를 준비해야 합니다:
  - SPO `cold.skey`.
  - 유효한 UTXO와 이에 연결된 `payment.vkey` 및 `payment.skey`.
  
  NEW: Cardano *extended* BIP32-Ed25519가 이제 지원됩니다!

- Partner-chain 의존성이 완전히 동기화되어 있어야 합니다 [(Step 2)](./step-2):
  - `Ogmios`와 `Postgres` 포트에 접근 가능해야 합니다.

:::tip

partner-chain 의존성이 있는 동일 인스턴스에서 등록을 수행하는 것이 더 쉽습니다. 특히 이 가이드에서 제공하는 compose 파일을 사용하는 경우에 그렇습니다.

:::

## 1. Install `midnight-node`.

Midnight-node에는 Midnight Partner-chain 컨트랙트와의 상호작용(예: block producer 후보 등록 제출)을 훨씬 쉽게 해주는 Partner-chain CLI wizard가 포함되어 있습니다.

:::note

현재 Midnight 프로젝트 단계에서 Midnight-node는 Docker 이미지로만 제공됩니다.

:::

1. `Midnight-node-docker` 저장소를 클론하세요(해당되는 경우):

:::tip

[(Step 2)](./step-2)에서 이미 `midnight-node-docker` 저장소를 클론했다면, 클론을 건너뛰고 해당 인스턴스를 사용하면 됩니다.

:::

```bash
git clone git@github.com:midnightntwrk/midnight-node-docker.git
cd midnight-node-docker
```

디렉토리에 들어가면, `direnv`가 `.envrc` 파일 허용 여부를 묻습니다.

## 2. Generate Partner-chain keys

이 시퀀스에서는 `midnight-shell.sh` 스크립트를 사용하여 컨테이너 내에서 `wizards` 명령을 실행합니다.

1. `midnight-node` 셸에 진입하세요:

```shell
./midnight-shell.sh
```

`./data` 디렉토리를 마운트하고 `midnight-node` 컨테이너 안으로 들어가서 `midnight-node` 명령을 실행할 수 있습니다.

예시:

```
$ ./midnight-shell.sh
root@dcb8af95f166:/# 
```

위 예시에서 `dcb8af95f166`은 midnight-node의 컨테이너 ID입니다. `midnight-shell.sh` 스크립트로 컨테이너에 진입하거나, docker 명령으로 직접 진입할 수도 있습니다:

```bash
docker container list -la # midnight-node 컨테이너와 "container ID" 확인
docker exec -it <container-id> /bin/bash # 컨테이너 ID로 셸 진입
docker exec -it dcb /bin/bash # 예: 컨테이너 "dcb8af95f166"에 진입
```

## 3. Make Cardano-keys accessible to container

등록은 이전 `generate-keys` 단계와 마찬가지로 midnight-node 컨테이너 내에서 수행됩니다. 지금 시점에서 필요한 Cardano 키가 컨테이너에서 접근 가능한지 확인해 두는 것이 좋습니다.

예를 들어, 호스트 머신의 `cardano-keys` 디렉토리에 키가 있다고 가정합니다. `docker cp` 명령으로 컨테이너에 복사할 수 있습니다.

```
docker cp /path/to/cardano-keys/ <container-ID>:cardano-keys
```

이 명령은 `cardano-keys` 디렉토리와 그 내용을 `midnight-node` 컨테이너의 루트 디렉토리에 배치합니다.

예시:

```bash
$ docker cp cardano-keys/ cb5:cardano-keys/
Successfully copied 49.2kB to cb5:cardano-keys/

$ docker exec -it cb5 /bin/bash

root@cb52f4691afd:/# ls
artifacts-amd64  data  lib	      mnt		    res   srv  var
bin		 dev   lib64	      opt		    root  sys  version
boot		 etc   media	      pc-chain-config.json  run   tmp
cardano-keys	 home  midnight-node  proc		    sbin  usr
```

컨테이너에서 이 디렉토리를 삭제하려면: 

```
docker exec <container-ID> rm -r cardano-keys/
```

## 3. Register as a candidate

등록은 3단계 명령 과정입니다. 다음을 수행해야 합니다: 

1. 등록 프로세스를 시작하고,
2. 등록 서명을 준비한 후,
3. 최종 등록을 제출합니다.

이 시퀀스는 `./midnight-node wizards register1`, `register2`, `register3` 명령으로 간소화됩니다.

<details>
<summary><b>Common Errors</b></summary>

1. **오류: "No UTXOs found for the given address"**
   - **원인:**
     - `payment.vkey`에서 파생된 주소에 미사용 트랜잭션 출력(UTXO)이 없습니다. 즉, testnet Ada(tAda)가 입금되지 않은 상태입니다.
     - Cardano 노드가 완전히 동기화되지 않아 최신 잔액을 조회하지 못할 수 있습니다.
   - **해결 방법:**
     - **주소에 자금 입금:** faucet(예: Cardano testnet faucet) 또는 다른 자금이 있는 지갑에서 payment 주소로 tAda를 전송하세요.
     - **노드 동기화 확인:** 다음 명령으로 노드가 완전히 동기화되었는지 확인하세요:
       ```bash
       ./cardano-cli.sh query tip
       ```
       `syncProgress`가 `100%`인지 확인하세요. 그렇지 않으면 동기화가 완료될 때까지 기다리세요.
</details>
