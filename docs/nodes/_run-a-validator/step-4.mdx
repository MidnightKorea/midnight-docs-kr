# Run a Midnight node in Validator Mode

## Prerequisites

- `Postgres` PORT가 `midnight-node`에서 접근 가능해야 하며, `cardano-db-sync`가 완전히 동기화되어 있어야 합니다 (Step 2).

- `Midnight-node`는 *현재* Docker 이미지로만 제공됩니다. 따라서 Docker / Docker Compose가 필요합니다.
    - [Docker / Docker Compose 설치하기](https://docs.docker.com/compose/install/).

- `data/chains/partner_chains_template/...` (Step 3에서 생성)이 Midnight-node에서 사용 가능해야 합니다(docker volume을 통해).
- 환경 변수 관리를 위해 `direnv`를 설치하세요.

`data/chains/partner_chains_template/...`이 아직 partner-chains deps 서버에 있다면, Midnight-node 인스턴스로 복사하세요.

<details>
<summary><b>선택 사항: Magic Wormhole을 사용한 인스턴스 간 파일 복사</b></summary>

설치
```shell
sudo apt install magic-wormhole # 각 인스턴스에 magic wormhole 설치

wormhole send data/ # 다른 인스턴스에서 복사하여 붙여넣을 receive 명령을 반환

# midnight-node용 인스턴스에서 receive 명령을 복사

wormhole receive <secret words>

# 디렉토리와 내용이 복사되었는지 확인
sudo apt install tree
tree data/
```
</details>

## Clone the `midnight-node-docker` repo and setup Partner-chain keys

- [midnight-node-docker](https://github.com/midnightntwrk/midnight-node-docker)를 클론하세요
- `data/chains/partner_chains_template/...`을 프로젝트 루트 디렉토리에 다음과 같이 복사하세요:

```shell
midnight-node-docker$ tree -d data/
data/
└── data
    └── chains
        └── partner_chains_template
            ├── keystore
            └── network
```

**`docker-compose up`으로 midnight-node 실행**:

- `compose.yml`이 있는 동일 디렉토리에서 `docker compose up`을 실행하여 노드를 시작하세요.

    ```shell
    docker-compose up
    # docker-compose up -d 로 분리 모드 실행
    ```

노드가 testnet과 동기화를 시작합니다.

```shell
⚙️ Syncing  2.6 bps, target=#474962 (8 peers), best: #3967 (0xf620…d752), finalized #3584 (0x56d5…f7bd), ⬇ 0.8kiB/s ⬆ 0.2kiB/s
```

위 로그 예시에서 노드는 동기화 중이며 네트워크와 피어링(8개)되어 있습니다. 현재 동기화된 블록은 3967이지만, 동기화 목표는 474962입니다.

:::note

등록 이후 n + 2 epoch부터 블록 생성 자격이 생깁니다. 예를 들어, epoch 900에 등록하고 epoch 901에 validator 노드를 시작하면, epoch 902가 될 때까지 블록이 생성되지 않습니다.

:::

노드가 완전히 동기화되면 블록 생성에 참여하게 됩니다. 예시:

```shell
✨ Imported #12345 (0xabc1…def2)
💤 Idle (5 peers), best: #12345 (0xabc1…def2), finalized #12343 (0x789a…bcd3)
🏆 Prepared block for proposing at #12346 (block time = 6s)
🔖 Pre-sealed block for proposal at #12346. Hash: 0xdef3…ghi4
✨ Successfully proposed block #12346 (0xdef3…ghi4)
✨ Imported #12346 (0xdef3…ghi4)
💤 Idle (5 peers), best: #12346 (0xdef3…ghi4), finalized #12344 (0x456b…def7)
```

- "Prepared block for proposing": 노드가 특정 블록 높이(예: #12346)에서 제안할 블록을 준비했다는 뜻입니다. 노드가 블록 생성에 적극적으로 참여하고 있다는 강력한 신호입니다.
- "Pre-sealed block for proposal": 해시를 포함하여 블록이 네트워크에 제안될 준비가 되었다는 뜻입니다.
- "Successfully proposed block": 노드가 블록을 네트워크에 성공적으로 제안했다는 뜻입니다.
- "Imported": 노드가 생성한 블록(또는 해당 높이의 다른 블록)이 체인에 가져와졌다는 뜻입니다. 해시가 제안한 블록과 일치하면, 노드의 블록이 수락되었다는 명확한 신호입니다.
- Block Number Incrementing: 최상위 블록 번호가 계속 증가하며 제안한 블록과 일치하면, 노드가 체인 진행에 기여하고 있는 것입니다.
