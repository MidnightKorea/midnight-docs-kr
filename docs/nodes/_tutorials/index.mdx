## Overview

Cardano Stake Pool Operator는 Midnight Testnet과 같은 Partner Chain의 블록 생성에 핵심적인 역할을 합니다. Midnight Testnet은 Cardano의 Preview Testnet과 연결되어 있습니다. 이 문서에서는 각각 Preview와 Testnet으로 지칭합니다.

### Environment setup

이 워크숍은 Multipass를 사용하여 가상 머신을 관리하는 단일 머신에서 실행하도록 설계되었습니다. 프로덕션 환경에서는 전용 베어메탈 서버나 클라우드 서비스를 선택할 수도 있습니다.

### Minimum system requirements

이 실습은 로컬 워크스테이션이나 전용 서버에서 진행할 수 있습니다.

#### Optimal Production Setup

이 튜토리얼은 간소화된 설정을 다루며, 프로덕션용 Cardano Stake Pool Operator(SPO) 구성 전체를 다루지는 않습니다. 프로덕션 환경에서 최적의 Cardano SPO 구성은 일반적으로 block producer 노드를 위한 전용 인스턴스와 다양한 지리적 위치에 배포된 여러 relay 노드를 포함합니다. 이러한 분산 아키텍처는 이중화를 보장하고 지연 시간을 최소화하여 네트워크 복원력, 보안, 성능을 향상시킵니다.

![cloud overivew](/img/spo_overview_ideal.png)

- mainnet 및 프로덕션용 공식 권장 시스템 요구사항은 Cardano 문서를 참조하세요: https://developers.cardano.org/docs/operate-a-stake-pool/hardware-requirements/. 

### Mac/ ARM compatibility:

- Cardano 도구의 지원이 더 우수한 AMD 또는 Intel 아키텍처를 사용하는 것을 강력히 권장합니다. ARM 및 M-Chip 아키텍처에 대한 사전 빌드 바이너리 지원은 제한적입니다:
    - ARM 또는 M-Chip 아키텍처를 사용하는 경우, 소스에서 직접 컴파일하거나 신뢰할 수 있는 릴리스를 사용해야 할 수 있습니다. 
    - ARM 및 M-Chip 사용자를 위한 선택적 해결 방법이 이 문서에서 제공됩니다.

### Windows compatibility:

    - Windows 사용자는 스크립트 실행을 위해 GitBash와 같은 Bash 호환 도구가 필요합니다.

### Prerequisites

    - System Administration: 시스템 리소스와 설정 관리에 익숙해야 합니다.
    - Networking: blockchain 노드 운영에 관련된 네트워크 토폴로지와 구성에 대한 이해가 필요합니다.
    - Command-Line Interface (CLI): CLI 도구 사용에 능숙해야 합니다. 설정의 상당 부분이 CLI를 통해 이루어집니다.

## Step 01 - Creating virtual machines using Multipass

### Introduction to Multipass

Multipass는 Ubuntu 가상 머신을 간편하게 실행하는 데 사용됩니다. Multipass를 설치하려면 Multipass 웹사이트에서 설치 프로그램을 다운로드하세요. Linux 사용자는 snap 패키지를 통해서도 설치할 수 있습니다:

#### Installation on Linux:

Multipass는 Ubuntu 가상 머신 인스턴스를 관리하기 위한 기본 hypervisor로 KVM(Kernel-based Virtual Machine)을 사용합니다. 따라서 KVM이 활성화되어 있는지 확인하는 것이 중요합니다.

- KVM이 활성화되어 있는지 확인하세요:

```shell
lsmod | grep kvm #check if KVM is enabled
```

- 출력 예시:

```shell
lsmod | grep kvm
kvm_intel             487424  0
kvm                  1404928  1 kvm_intel
irqbypass              12288  1 kvm
```

- Snap이 설치되어 있지 않다면 설치하세요:

```shell
sudo apt install snapd
```

- Multipass 설치:

```shell
sudo snap install multipass --classic
```

- 설치 확인:

```shell
multipass --version # 현재 버전 표시
multipass --help # 사용법 및 명령 표시
which multipass # 설치 경로 확인
```

### Launch virtual machines

- 2개의 가상 머신을 생성하려면 다음 스크립트를 사용하세요:

```shell
#!/bin/bash

# Launch multipass instances
echo "✨ Creating cute little instances! ✨"
multipass launch -n cn1 -c 2 -m 4gb -d 40gb
multipass launch -n cn2 -c 2 -m 4gb -d 40gb
echo "🐾 Instances are born! 🐾"

instances=("cn1" "cn2")

# Function to check if architecture is ARM
is_arm_architecture() {
    if uname -m | grep -q "arm" || uname -m | grep -q "aarch64"; then
        return 0  # True (ARM detected)
    else
        return 1  # False (not ARM)
    fi
}

for instance in "${instances[@]}"; do
    echo "🚀 Running commands on $instance... 🌟"

    multipass exec "$instance" -- bash -c '
        # Always run the standard setup first
        echo "🐼 Running standard setup for all architectures... 🐼"
        sudo apt update -y && sudo apt upgrade -y
        mkdir -p "$HOME/tmp" && cd "$HOME/tmp"
        sudo apt -y install curl
        echo "🐸 Downloading guild-deploy script..."
        curl -sS -o guild-deploy.sh https://raw.githubusercontent.com/cardano-community/guild-operators/master/scripts/cnode-helper-scripts/guild-deploy.sh
        chmod 755 guild-deploy.sh
        ./guild-deploy.sh -b master -n preview -t cnode -s pdlcowx
        . "${HOME}/.bashrc"
        cd ~/git || mkdir -p ~/git && cd ~/git
        echo "🦋 Cloning cardano-node repository..."
        git clone https://github.com/intersectmbo/cardano-node || (cd cardano-node && git fetch --tags --recurse-submodules --all && git pull)
        cd cardano-node
        git checkout $(curl -sLf https://api.github.com/repos/intersectmbo/cardano-node/releases/latest | jq -r .tag_name)
        $CNODE_HOME/scripts/cabal-build-all.sh

        # Check architecture and run additional ARM-specific steps if needed
        if uname -m | grep -q "arm" || uname -m | grep -q "aarch64"; then
            # ARM-specific additional commands
            echo "🐙 Detected ARM architecture, running additional ARM-specific setup... 🐙"
            ARCHIVE_URL="https://github.com/armada-alliance/cardano-node-binaries/raw/refs/heads/main/static-binaries/cardano-10_1_4-aarch64-static-musl-ghc_966.tar.zst"
            ADDRESS_URL="https://github.com/armada-alliance/cardano-node-binaries/raw/refs/heads/main/miscellaneous/cardano-address.zip"
            DEST_DIR="$HOME/.local/bin"
            mkdir -p "$DEST_DIR"
            
            echo "🐳 Downloading cardano-node archive..."
            curl -L "$ARCHIVE_URL" -o cardano-10_1_4.tar.zst
            if [ $? -ne 0 ]; then
                echo "😿 Failed to download the cardano-node archive."
                exit 1
            fi
            
            echo "🦄 Downloading cardano-address archive..."
            curl -L "$ADDRESS_URL" -o cardano-address.zip
            if [ $? -ne 0 ]; then
                echo "😿 Failed to download the cardano-address archive."
                exit 1
            fi

            if ! command -v zstd &> /dev/null; then
                sudo apt-get update -y
                sudo apt-get install -y zstd unzip
            elif ! command -v unzip &> /dev/null; then
                sudo apt-get install -y unzip
            fi

            echo "🐰 Extracting cardano-node archive..."
            zstd -d -c cardano-10_1_4.tar.zst | tar -x -C "$DEST_DIR" --strip-components=1
            if [ $? -ne 0 ]; then
                echo "😿 Failed to extract the cardano-node archive."
                exit 1
            fi

            echo "🦊 Extracting cardano-address archive..."
            unzip -d "$DEST_DIR" cardano-address.zip
            if [ $? -ne 0 ]; then
                echo "😿 Failed to extract the cardano-address archive."
                exit 1
            fi

            rm cardano-10_1_4.tar.zst cardano-address.zip
            echo "🌈 Additional ARM archives downloaded and extracted successfully to $DEST_DIR"
        fi
    '

    echo "🎉 Finished running commands on $instance! 🐾✨"
done
```

이 스크립트는 cn1과 cn2("cardano-node-1"과 "cardano-node-2")라는 이름의 VM 2개를 생성합니다. 각각 CPU 2코어, 메모리 4 GB, 디스크 스토리지 20 GB를 할당합니다.

#### Listing instances

- 실행 중인 모든 인스턴스를 나열하여 설정을 확인하세요:

```shell

```

출력 예시:



:::

#### Graphical Interface

## Step 02 - Configure and start relay rode (cn1)

### Access relay node instance

```shell
multipass shell cn1
```

### Initial setup verification

- 설치 시 SPO toolkit이 `$CNODE_HOME` 변수를 Cardano 노드 파일이 저장되는 `/opt/cardano/cnode`로 설정했을 것입니다.
- 설정을 확인하세요:
    - echo `$CNODE_HOME`이 `/opt/cardano/cnode`를 반환해야 합니다.
    - which cardano-node이 `/home/ubuntu/.local/bin/cardano-node`를 반환해야 합니다.

### Environment variables

SPO toolkit은 `$CNODE_HOME/scripts/env`에 기본 경로와 변수를 제공합니다. 주요 변수는 다음과 같습니다:

```shell
CNODEBIN: "/home/ubuntu/.local/bin/cardano-node"
CCLI: "/home/ubuntu/.local/bin/cardano-cli"
CNODE_PORT: 6000
POOL_NAME: "GUILD"
```

환경 변수를 재정의하여 원하는 대로 커스터마이징할 수 있습니다. 예: `export CNODE_PORT=6001`.

### Test start the relay node

- scripts 디렉토리로 이동하세요:

```shell
cd "${CNODE_HOME}"/scripts
```

- 셸에서 노드를 대화형으로 시작하세요:

```shell
./cnode.sh
```

- 출력이 `Listening on http://127.0.0.1:12798`로 끝나야 합니다.
- Ctrl + C를 눌러 노드를 중지하세요.

### Connect relay node (cn1) to block producer node (cn2)

relay 노드(cn1)는 전체 네트워크와 피어링됩니다. 반면 block producer 노드(cn2)는 relay 노드(cn1)하고만 피어링됩니다. cn1 설정 파일과 토폴로지를 수정하면 됩니다.

![relay-bp-relation](/img/spo_tutorial-relay-bp-relation.jpg)

- `config.json`을 편집하여 peer sharing을 활성화하세요:

```shell
vim $CNODE_HOME/files/config.json
```

"PeerSharing"을 false에서 true로 변경하세요.

- block producer 노드에 대한 지속 연결을 위해 topology.json을 수정하세요:
    - $CNODE_HOME/files/topology.json을 열고 "localRoots"의 "accessPoints"에 block producer 노드(cn2)를 추가하세요. 플레이스홀더 "replace-this-with-BP"를 cn2의 IP와 포트로 교체하세요. 추가 relay 연결이 필요하지 않으면 relay 줄을 삭제하세요.
    - block producer 노드가 하나뿐이므로 "hotValency"를 1로 설정하세요.

예를 들어, 다음을: 

```shell
"localRoots": [
    {
      "accessPoints": [
        {"address": "127.0.0.1", "port": 6000, "description": "replace-this-with-BP"},
        {"address": "127.0.0.1", "port": 6001, "description": "replace-this-with-relay"}
      ],
      "advertise": false,
      "trustable": true,
      "hotValency": 2
    },
```

다음으로 변경하세요: 

```shell
  "localRoots": [
    {
      "accessPoints": [
        {"address": "x.x.x.x", "port": 6000, "description": "cn2 BP node"}
      ],
      "advertise": false,
      "trustable": true,
      "hotValency": 1
    },
```

전체 `topology.json` 예시:

### Configure relay node to run as a systemd service for automatic startup

Cardano 노드를 수동으로 시작하는 방식에서 자동화된 프로세스로 전환하려면, systemd 서비스로 구성하세요. 이 설정은 자동 시작과 시스템 통합을 지원합니다. 진행 방법은 다음과 같습니다:

- $CNODE_HOME/scripts 디렉토리로 이동하세요:

```shell
cd $CNODE_HOME/scripts
```

- 노드 서비스를 위한 systemd 설정 스크립트를 실행하세요:

```shell
./cnode.sh -d
# Deploying cnode.service as systemd service
# cnode.service deployed successfully
```

- submit-api 서비스를 위한 systemd 설정 스크립트를 실행하세요:

```shell
./submitapi.sh -d
# Deploying cnode-submit-api.service as systemd service
# cnode-submit-api deployed successfully
```

- systemd 서비스를 시작하세요:

```shell
sudo systemctl start cnode.service
sudo systemctl start cnode-submit-api.service
```

- 서비스 상태를 확인하고 관리하세요:

```shell
sudo systemctl status cnode.service
sudo systemctl status cnode-submit-api.service
```

필요에 따라 status를 `stop`, `start`, `restart`로 교체하세요.

- 선택적으로 gLiveView로 노드 서비스를 모니터링하세요.

```shell
cd $CNODE_HOME/scripts
./gLiveView.sh
```

이 시점에서 cn1과 cn2가 완전히 동기화될 때까지 기다려야 합니다. 몇 시간이 걸릴 수 있습니다.

## Step 03 - Start cn2 as a relay node

cn1은 relay 노드로 지정되었습니다. cn2는 처음에 relay 노드로 설정하지만, 나중에 block producer 노드로 전환합니다.

### Access BP node instance

```shell
multipass shell cn2
```

### Configure relay rode to run as a systemd service for automatic startup

Cardano 노드를 수동 시작에서 자동화된 프로세스로 전환하려면, systemd 서비스로 구성하세요. 이 설정은 자동 시작과 시스템 통합을 지원합니다. 진행 방법은 다음과 같습니다:

- `$CNODE_HOME/scripts` 디렉토리로 이동하세요:

```shell
cd $CNODE_HOME/scripts
```

- 노드 서비스를 위한 systemd 설정 스크립트를 실행하세요:

```shell
./cnode.sh -d
# Deploying cnode.service as systemd service
```
