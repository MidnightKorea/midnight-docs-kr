# Compact compiler 0.19.0 (Compact language 0.11.0)

Compact은 Midnight의 전용 스마트 컨트랙트 프로그래밍 언어로, 안전하고 효율적이며 유연한 탈중앙화 애플리케이션을 구축하기 위해 설계되었습니다.

Compact 컴파일러는 Compact 스마트 컨트랙트 코드를 실행 가능한 바이트코드와 증명 생성 회로로 변환합니다.

## Compact compiler 0.19.0 (Compact language 0.11.0) release notes

오늘 Compact 컴파일러 버전 0.19.0을 릴리스합니다. 이 릴리스는 Compact 언어 버전 0.11.0을 지원하며, Compact 버전 0.10.x에서 일부 호환성을 깨는 변경이 포함되어 있습니다. 이러한 변경은 Compact 언어를 TypeScript와 더 유사하게 만들고 향후 언어 기능을 위한 기반을 마련하기 위한 것입니다.

새 릴리스를 다운로드하여 사용해 볼 수 있습니다. 새 버전의 컴파일러로 Compact 코드를 다시 컴파일하면 아래 설명에 따라 코드를 업데이트해야 합니다. `compactc` 0.19.0으로 반드시 업그레이드할 필요는 없습니다. Compact 코드를 업데이트하지 않고 `compactc` 0.18.x를 계속 사용해도 정상 동작합니다. 다만, `compactc` 0.18.x에 대한 추가 버그 수정 릴리스는 계획하고 있지 않습니다.

## Summary of Changes

- 제네릭이 이제 꺾쇠괄호를 사용합니다.
- 제네릭의 크기 매개변수에 `#` 접두사가 붙습니다.
- `Unsigned Integer`가 `Uint`로 변경되었습니다.
- 범위 제한 부호 없는 정수의 구문이 새로워졌습니다.
- 기본값을 위한 `null()`이 `default<>`로 변경되었습니다.
- `map`과 `fold` 표현식의 구문이 변경되었습니다.

---

## Generics Now Use Angle Brackets

Compact의 제네릭은 선언(`struct Foo[n, T] {...}`)과 사용(`Foo[3, Field]`) 모두에서 대괄호를 사용했습니다. 이를 꺾쇠괄호로 변경합니다. 이제 `struct Foo<n, T> {...}`와 `Foo<3, Field>`로 작성합니다.

이것은 **호환성을 깨는 변경**입니다. 이전 구문은 더 이상 동작하지 않습니다.

### Why We Made This Change

대괄호는 역사적인 이유로 존재했으며, TypeScript와 다르게 할 타당한 이유가 없습니다. 이 차이는 TypeScript 개발자가 Compact 코드를 읽기 어렵게 만들고, 두 언어 간 전환을 불편하게 합니다.

### How to Fix Your Code

제네릭 선언과 사용에서 대괄호 `[]`를 꺾쇠괄호 `<>`로 변경하세요. 내장 타입인 `Vector`, `Bytes`, `Unsigned Integer`(아래 타입 이름 변경 참조), `Opaque`에도 적용됩니다. 변경하지 않으면 컴파일러가 구문 오류를 표시합니다.

Compact에서 벡터 생성 표현식이나 벡터 인덱싱에 사용되는 대괄호는 변경하지 않도록 주의하세요. 실수로 변경하면 컴파일러가 구문 오류를 표시합니다.

---

## Size Parameters in Generics Are Prefixed with `#`

Compact에는 두 종류의 제네릭 매개변수가 있습니다: 타입(Compact 타입으로 인스턴스화됨)과 크기(리터럴 정수로 인스턴스화됨). 제네릭 선언에서 크기 매개변수 앞에 번호 기호(`#`)를 붙이도록 구문을 변경합니다. 타입 매개변수에는 접두사가 없습니다. 예를 들어:

```compact
struct Foo<#m, #n, T> {
  b: Bytes<m>;
  v: Vector<n, T>;
}
```

이것은 **호환성을 깨는 변경**입니다. 번호 기호는 이제 제네릭 크기 매개변수 선언에 필수이며, 누락 시 구문 오류가 발생합니다.

### Why We Made This Change

이전 구문에서는 제네릭 인수가 타입인지 숫자인지 알기 위해 제네릭 선언의 전체 본문을 읽어야 했습니다. 제네릭 크기 매개변수에 주석을 달면 선언의 첫 번째 줄만 읽어도 각 매개변수의 종류를 알 수 있습니다.

이 변경은 향후 더 나은 오류 보고도 가능하게 합니다. 컴파일러가 선언 자체에서 선언 오류를 표시할 수 있으며, 잘못된 선언을 허용하고 사용처에서 원격으로 오류를 표시하는 대신 직접 표시할 수 있습니다.

### How to Fix Your Code

제네릭 선언을 검사하고 크기 매개변수에 번호 기호 접두사를 추가하세요. 크기 매개변수에 주석을 달지 않으면, 해당 제네릭이 크기로 인스턴스화되는 모든 곳에서 컴파일러가 오류를 표시합니다.

---

## `Unsigned Integer` Is Changed to `Uint`

Compact 부호 없는 정수 타입의 철자를 변경했습니다. 이전에는 `Unsigned Integer`였으나 이제 `Uint`로 변경되었습니다.

이것은 **호환성을 깨는 변경**입니다.

### Why We Made This Change

`Unsigned Integer`는 자주 사용되는 타입치고는 너무 길었습니다. Compact 코드를 읽고 쓰기 불편하게 만들었습니다. 또한 `Unsigned`가 수정자처럼 보이지만, `Integer`만으로는 타입이 아니어서 혼동을 줄 수 있었습니다.

### How to Fix Your Code

`Unsigned Integer`를 `Uint`로 바꾸세요. 검색-치환으로 처리할 수 있지만, `Unsigned`와 `Integer` 사이에 비표준 공백을 넣은 경우 모든 항목을 찾도록 주의하세요.

---

## Bounded Unsigned Integers Have New Syntax

Compact에는 두 종류의 부호 없는 정수 타입이 있습니다: 크기 타입(최대값을 표현하는 데 필요한 비트 수로 매개변수화됨, 예: `Uint<32>`)과 범위 제한 타입(상한으로 매개변수화됨, 예: `Unsigned Integer[<= 99]`). 범위 제한 정수 타입의 구문을 하한과 상한 모두 포함하도록 변경합니다. 현재 하한은 반드시 `0`이어야 합니다. 두 경계 모두 포함(inclusive)입니다.

위의 범위 제한 타입은 이제 다음과 같이 작성합니다:

```compact
Uint<0..99>
```

이것은 **호환성을 깨는 변경**입니다. 이전 "작거나 같음" 구문은 더 이상 동작하지 않습니다.

### Why We Made This Change

제네릭을 꺾쇠괄호로 변경한 후, `Uint<<= 99>`가 이중 왼쪽 꺾쇠괄호로 인해 읽고 쓰기 불편했습니다. 또한 향후 하한을 위한 명확하고 간결한 확장을 지원하는 구문을 원했습니다.

### How to Fix Your Code

범위 제한 정수 타입에서 `<=`를 `0..`으로 바꾸세요.

---

## `null()` for Default Values Is Now `default<>`

Compact의 모든 타입에는 해당 타입의 기본값이 있습니다. `Foo` 타입의 기본값은 이전에 `null(Foo)`로 작성했습니다. 이 구문을 `default<Foo>`로 변경했습니다.

이것은 **호환성을 깨는 변경**입니다. `null`은 더 이상 동작하지 않습니다.

### Why We Made This Change

Compact 타입의 기본값은 TypeScript의 `null`과 매우 다릅니다. Compact에서는 모든 타입에 기본값(해당 타입의 값)이 있습니다. TypeScript에서 `null`은 별개의 `Null` 타입의 유일한 값입니다. 이 변경으로 혼동을 피하고 Compact의 구문을 TypeScript에 더 가깝게 맞춥니다.

### How to Fix Your Code

`null(...)`을 `default<...>`로 바꾸세요.

---

## `map` and `fold` Expressions Have Different Syntax

Compact에는 벡터를 순회하는 `map`과 `fold` 키워드가 있습니다. 두 인수 회로를 한 쌍의 벡터에 매핑하는 이전 구문은 다음과 같았습니다:

```compact
map add over v0 over v1
```

마찬가지로, 초기값과 함께 두 인수 회로를 벡터에 폴딩하는 구문은 다음과 같았습니다:

```compact
fold add 0 over v
```

구문을 함수 호출처럼 보이도록 변경했습니다. 위의 예시는 이제 다음과 같이 작성합니다:

```compact
map(add, v0, v1)
fold(add, 0, v)
```

이것은 **호환성을 깨는 변경**입니다.

### Why We Made This Change

`map`과 `fold` 표현식이 읽고 쓰기 불편했습니다. 새 구문은 인수를 더 명확하게 하고 TypeScript와 더 잘 맞습니다.

### How to Fix Your Code

`map`과 `fold` 표현식이 인수에 괄호와 쉼표를 사용하도록 하세요. `over` 키워드 사용을 제거하세요.

---

## TL;DR

Compact 언어 0.11.0 릴리스는 TypeScript와 더 가깝게 맞추는 호환성을 깨는 구문 변경을 도입합니다:

- **제네릭 구문**: 대괄호 대신 꺾쇠괄호 `<>`를 사용합니다.
- **크기 매개변수**: 제네릭 선언에서 `#` 접두사를 붙입니다.
- **타입 이름 변경**: `Unsigned Integer`가 `Uint`로 변경되었습니다.
- **범위 제한 정수 구문**: 하한과 상한 모두 포함, 예: `Uint<0..99>`.
- **기본값 구문**: `null(Foo)`에서 `default<Foo>`로 변경되었습니다.
- **`map`과 `fold` 구문**: 함수 호출 형식으로 변경되었습니다.

이러한 업데이트로 인해 새 버전에서 컴파일하려면 코드 수정이 필요합니다.
