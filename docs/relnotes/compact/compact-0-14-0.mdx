# Compact compiler 0.21.0 (Compact language 0.14.0)

Compact은 Midnight 전용 스마트 컨트랙트 언어로, 안전하고 효율적인 탈중앙화 애플리케이션 개발을 위해 만들어졌습니다.

Compact 컴파일러는 스마트 컨트랙트 코드를 실행 가능한 바이트코드와 증명 생성 회로로 변환합니다.

## Compact compiler 0.21.0 (Compact language 0.14.0) release notes

Compact 컴파일러 버전 0.21.0을 릴리스합니다. 언어 버전이 0.14.0으로 올라가며, 0.13.x 대비 다수의 개선 사항과 호환성을 깨는 변경이 포함되어 있습니다. 대규모 업데이트이므로 릴리스 노트를 꼼꼼히 확인해 주세요.

새 컴파일러로 기존 Compact 코드를 다시 컴파일하면, 호환성을 깨는 변경에 따라 코드 수정이 필요할 수 있습니다.

## Summary of Changes

### Language Changes

- **[호환성을 깨는 변경]** 위트니스 반환 값에 `disclose`(공개)가 필요할 수 있습니다.
- 대괄호로 TypeScript 호환 튜플을 생성할 수 있습니다.
- TypeScript 호환 구조 분해를 지원합니다.
- **[호환성을 깨는 변경]** `Void` 타입이 제거되었습니다.
- **[호환성을 깨는 변경]** `for` 루프 구문이 TypeScript에 맞춰졌습니다.
- **[호환성을 깨는 변경]** `Cell` 원장 ADT가 이제 암시적입니다.
- **[호환성을 깨는 변경]** 원장 ADT 이름이 이제 표준 라이브러리의 일부입니다.
- 내보낸 회로가 위트니스와 동일한 이름을 가질 수 있습니다.
- 구문 오류 메시지가 개선되었습니다.

## Language Changes

### **[Breaking Change]** Witness Return Values May Need to Be "Disclosed"

Compact 0.14.0부터 위트니스 반환 값은 비공개 데이터로 취급됩니다. 퍼블릭 원장에 기록되는 등 의도하지 않은 데이터 유출을 방지하기 위해, 컴파일러가 `disclose` 구문을 통한 명시적 공개를 요구합니다.

#### Example:

```compact
// 0.14.0 이전 버전:
poster = public_key(local_secret_key(), instance as Field as Bytes<32>);

// 0.14.0 이후 버전 (공개 포함):
poster = disclose(public_key(local_secret_key(), instance as Field as Bytes<32>));
```

#### Why We Made This Change

비공개 데이터를 개발자가 의도적으로만 공개하도록 보장하기 위함입니다.

#### How to Fix Your Code

비공개 데이터를 의도적으로 공개할 때 위트니스 반환 값을 `disclose`로 감싸세요.

---

### Square Brackets Construct TypeScript-Compatible Tuples

대괄호가 이제 벡터 대신 튜플을 생성합니다. 튜플은 각 요소의 타입을 정확히 추적하며, TypeScript와 호환됩니다.

#### Example:

```compact
// 튜플 예시:
const tuple = [1, 2, 3]; // 타입: [Uint<0..0>, Uint<0..1>, Uint<0..2>]

// 튜플 서브타이핑 예시:
const vector: Vector<3, Uint<8>> = tuple;
```

벡터 타입은 여전히 존재합니다. 동일한 요소 타입을 가진 튜플은 벡터와 동등합니다.

---

### You Can Use TypeScript-Compatible Destructuring

const 바인딩과 회로 매개변수(화살표 회로 포함)에서 튜플, 벡터, 구조체에 대한 구조 분해를 지원합니다.

#### Example:

```compact
const [x, y] = [1, 2];

circuit example([a, b]: [Uint<8>, Uint<8>]): Uint<8> {
  return a + b;
}
```

#### Limitations:

- `...`(스프레드) 구문은 아직 지원되지 않습니다.
- 원장 필드 할당에서는 구조 분해가 동작하지 않습니다.

---

### **[Breaking Change]** The `Void` Type Is Removed

`Void` 타입이 빈 튜플 `[]`로 대체되었습니다. 반환 값이 없는 회로나 위트니스는 이제 `[]`를 반환해야 합니다.

#### Example:

```compact
// 이전 구문:
circuit foo(): Void {
  return;
}

// 새 구문:
circuit foo(): [] {
  return [];
}
```

#### Why We Made This Change

`Void`의 동작이 직관적이지 않아, 더 단순하고 일관된 단위 타입으로 교체했습니다.

#### How to Fix Your Code

`Void`를 `[]`로 바꾸세요.

---

### **[Breaking Change]** The Syntax of `for` Loops Is Aligned with TypeScript

`for` 루프 구문이 이제 TypeScript와 일치합니다. 이전 Compact 전용 구문 대신 `for (const x of collection)`을 사용하세요.

#### Examples:

```compact
// 이전 구문:
for x in v do {
  // 루프 본문
}

// 새 구문:
for (const x of v) {
  // 루프 본문
}
```

#### Why We Made This Change

TypeScript와의 구문 차이를 줄이기 위함입니다.

#### How to Fix Your Code

`for` 루프를 새 구문으로 업데이트하세요.

---

### **[Breaking Change]** The `Cell` Ledger ADT Is Now Implicit

원장 필드에서 더 이상 `Cell`을 명시적으로 선언하지 않습니다. 대신 Compact이 암시적으로 셀로 취급합니다.

#### Example:

```compact
// 이전 구문:
ledger state: Cell<STATE>;

// 새 구문:
ledger state: STATE;
```

#### Why We Made This Change

원장 필드 선언을 간소화하기 위함입니다.

#### How to Fix Your Code

원장 필드 선언에서 `Cell`을 제거하세요.

---

### **[Breaking Change]** Ledger ADT Names Are Now Part of the Standard Library

원장 ADT 타입 이름(예: `Counter`, `Set`)은 더 이상 내장 식별자가 아닙니다. 이제 표준 라이브러리의 일부이며 명시적으로 가져와야 합니다.

#### Example:

```compact
import CompactStandardLibrary;

const counter: Counter = CompactStandardLibrary.Counter();
```

#### Why We Made This Change

더 유연한 이름 지정과 모듈화를 위함입니다.

#### How to Fix Your Code

원장 ADT를 사용하는 경우 표준 라이브러리를 가져오세요.

---

### Exported Circuits Are Allowed to Have the Same Name as Witnesses

내보낸 회로가 위트니스와 같은 이름을 가질 수 있습니다. 모호한 경우 컴파일러가 오류를 표시하지만, 대부분의 경우 수정 없이 동작합니다.

---

### Improved Syntax Error Messages

구문 오류 메시지에 예상 구문과 실제 발생한 문제가 함께 표시되어, 오류를 더 쉽게 파악하고 해결할 수 있습니다.

---

## TL;DR

Compact 언어 0.14.0의 주요 변경 사항입니다:

- **위트니스 공개**: 필요시 위트니스 반환 값에 `disclose`를 사용하세요.
- **튜플**: 대괄호가 이제 TypeScript 호환 튜플을 생성합니다.
- **구조 분해**: 튜플, 벡터, 구조체에 대한 TypeScript 호환 구조 분해.
- **`Void` 제거**: 빈 튜플 타입 `[]`을 대신 사용하세요.
- **`for` 루프**: TypeScript 구문에 맞춰졌습니다.
- **원장 필드**: `Cell`이 이제 암시적입니다.
- **원장 ADT**: 표준 라이브러리의 일부입니다.
- **오류 메시지**: 더 유용한 구문 오류 메시지.

위 변경 사항에 맞게 코드를 업데이트하세요.
