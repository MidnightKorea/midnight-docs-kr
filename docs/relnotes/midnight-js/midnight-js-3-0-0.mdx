## High-level summary

이번 릴리스는 Midnight.js에 새로운 암호화 저장소 보안 모델, 비동기 트랜잭션 처리, 크게 개선된 트랜잭션 워크플로를 포함한 주요 업그레이드를 제공합니다. v2.x에서 업그레이드하는 개발자는 호환성에 영향을 주는 변경 사항과 마이그레이션 단계를 신중하게 검토해야 합니다.

## Audience

이 릴리스 노트는 다음에 해당하는 개발자에게 중요합니다:

- Midnight.js을 사용하여 애플리케이션이나 스마트 컨트랙트를 구축하는 경우
- `LevelPrivateStateProvider`을 통해 private state 저장소를 관리하는 경우
- 동기식 트랜잭션 제출 또는 컨트랙트 호출에 의존하는 경우
- 인덱서와 통합하는 경우

## Summary of updates

- 설정 가능한 비밀번호 제공자를 포함한 암호화된 private state 저장소를 도입했습니다.
- 트랜잭션 제출과 컨트랙트 호출을 비동기 API로 전환했습니다.
- 명시적 트랜잭션 상태를 가진 새로운 증명 레시피 시스템을 추가했습니다.
- 상위 레벨 제출 헬퍼를 사용하여 트랜잭션 워크플로를 단순화했습니다.
- 비차폐(unshielded) NIGHT 토큰 조회를 지원합니다.
- 트랜잭션 TTL(유효 기간) 지원을 활성화했습니다.
- Compact 컴파일러 및 런타임 의존성을 업데이트했습니다.
- ESM 및 CommonJS 패키지 호환성을 개선했습니다.

## New features

아래는 추가된 기능의 상세 내역입니다.

### Storage encryption

Private state 저장소가 이제 명시적 인증 설정과 함께 AES-256-GCM 암호화를 사용합니다.

이 변경으로 보안이 향상되고 시크릿 매니저와의 통합이 가능해집니다.

### Configurable password provider

Midnight.js이 이제 암호화된 저장소를 위한 `PrivateStoragePasswordProvider`을 지원합니다. 이 개선을 통해 개발자는 다음을 할 수 있습니다:

- 저장소 암호화를 지갑에서 분리
- 커스텀 비밀번호 관리 활성화
- 보안 시크릿 매니저 지원
- 유연한 폴백 전략 제공

### Async transaction handling

트랜잭션 제출과 컨트랙트 호출이 이제 비동기 실행을 지원합니다.

이를 통해 다음이 가능합니다:
- 완료까지 무기한 대기
- 개선된 오류 처리
- 최신 비동기 워크플로와의 더 나은 정합성

모든 `submitTx` 및 `contract.call.*` 호출에 await을 사용해야 합니다.

### Enhanced proving recipe system

트랜잭션 증명 파이프라인이 세 가지 명시적 상태를 가진 새로운 `BalancedProvingRecipe` 유니온 타입으로 재설계되었습니다:

- `TransactionToProve`
- `BalanceTransactionToProve`
- `NothingToProve`

이를 통해 트랜잭션 상태가 명시적이 되어 제출 시 잘못된 가정을 방지합니다.

### High-level transaction functions

새로운 헬퍼 함수가 밸런싱, 증명, 제출을 내부적으로 처리하여 트랜잭션 라이프사이클을 단순화합니다:

- `submitDeployTx`
- `submitCallTx`

### Unshielded NIGHT token support

Midnight.js이 이제 공개 인덱서를 통해 비차폐 NIGHT 토큰의 잔액 조회를 지원합니다.

### Transaction TTL support

트랜잭션에 밸런싱 시 유효 기간(TTL)을 할당하여 만료된 제출을 방지하고 네트워크 리소스를 최적화할 수 있습니다.

### Compact compiler update

Compact 컴파일러 v0.27.0으로 업데이트되어 다음을 제공합니다:
- 개선된 타입 추론
- 더 나은 오류 메시지
- 향상된 회로 최적화

### ESM and CommonJS dual support

패키지 exports가 ESM과 CommonJS 환경을 모두 완전히 지원하도록 수정되어, 최신 번들러 및 빌드 시스템과의 호환성이 개선되었습니다.

### Binary circuit results support

전체 성능 향상을 위해 바이너리(Uint8Array) 회로 결과를 도입했습니다. 이점은 다음과 같습니다:
- 회로 실행 중 메모리 할당 감소
- 더 빠른 직렬화 및 역직렬화
- 대규모 또는 복잡한 회로 작업 시 성능 향상

## Breaking changes

다음 변경 사항은 기존 기능을 깨뜨리는 중요한 내용입니다.

### LevelPrivateStateProvider authentication required

**변경 내용**: 암묵적 저장소 인증이 제거되었습니다.

**영향**: `walletProvider` 또는 `privateStoragePasswordProvider` 없이 인스턴스를 생성하는 모든 코드에 영향을 줍니다.

**필요한 조치**:
- 모든 provider 인스턴스화를 업데이트하세요.
- 세션 간 비밀번호 일관성을 확보하세요.

### WalletProvider.balanceTx return type

**변경 내용**: `balanceTx`이 이제 `BalancedProvingRecipe` 유니온을 반환합니다.

**영향**: 단일 트랜잭션 타입을 가정하는 코드에 영향을 줍니다.

**필요한 조치**:
- 타입 판별 로직을 추가하세요.
- 세 가지 레시피 변형을 모두 명시적으로 처리하세요.

### MidnightProvider.submitTx is now async

**변경 내용**: 반환 타입이 `TransactionId`에서 `Promise<TransactionId>`로 변경되었습니다.

**영향**: 동기식 제출 로직에 영향을 줍니다.

**필요한 조치**:
- 모든 `submitTx` 호출에 `await`을 추가하세요.
- 호출하는 함수를 `async`로 업데이트하세요.

```typescript
// Before
const txId = midnightProvider.submitTx(tx);

// After
const txId = await midnightProvider.submitTx(tx);
```

### Contract calls are now async

**변경 내용**: 모든 `contract.call.*` 메서드가 이제 `Promise<T>`을 반환합니다.

**영향**: 동기식 컨트랙트 호출 사용에 영향을 줍니다.

**필요한 조치**:
- 모든 컨트랙트 호출에 await을 추가하세요.
- 의존하는 로직을 그에 맞게 업데이트하세요.

### Zswap offer generation

**변경 내용**: `zswapStateToOffer()`이 이제 `undefined`을 반환할 수 있습니다.

**영향**: 항상 offer가 생성된다고 가정하는 코드에 영향을 줍니다.

**필요한 조치**: 빈 Zswap 상태에 대한 검사를 추가하세요.

```typescript
const offer = zswapStateToOffer(state, encKey);
if (!offer) {
  // Handle empty state
}
```

### networkId type change

**변경 내용**: `networkId`이 enum에서 일반 문자열로 변경되었습니다.

**영향**: enum 임포트 및 패턴 매칭에 영향을 줍니다.

**필요한 조치**:
- `NetworkId` enum 사용을 제거하세요.
- 문자열 리터럴로 대체하세요 (예: `'preview'`).

```typescript 
// Before
import { NetworkId } from '@midnight-ntwrk/types';
config.networkId = NetworkId.Testnet;

// After
config.networkId = 'preview';
```

## Known issues

릴리스 시점에 보고된 치명적인 알려진 이슈는 없습니다. 문제가 발생하는 개발자는 다음을 확인하세요:
- Node.js 버전 22 이상이 설치되어 있는지 확인하세요.
- 컨트랙트 호출에 `await` 키워드가 추가되어 있는지 확인하세요.
