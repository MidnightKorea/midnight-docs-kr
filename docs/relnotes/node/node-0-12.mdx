## Midnight Node 0.12.0 Release Notes

### Stable Release

* **Version**: 0.12.0
* **Audience**: 노드 운영자, Midnight 생태계와 통합하는 개발자
* **Environment**: 프로덕션 준비 완료

0.12.0은 안정 릴리스로, 프로덕션 환경에 배포할 수 있습니다.

PC1.5 및 BLS 지원을 포함하며, 새로운 스토리지 모델을 사용합니다.

### Key Changes & Highlights

:::note
이 변경 사항은 프로덕션 환경(testnet-02)의 체인 리셋이 필요합니다.
:::

* 필드/커브를 Pluto-Eris에서 BLS12-381로 전환
  - 이유: 더 작고 빠른 증명. 공개 파라미터 크기 10배 감소(500MB에서 50MB로).

* 새로운 스토리지 모델로 전환
  - 원장 상태 크기에 영향을 받지 않는 더 빠른 성능

* PC1.5


### Changelog

#### Added

- DB Sync에서 제거된 누락 인덱스를 추가했습니다.
- 이미지에 더 많은 디버깅 도구를 기본 포함했습니다.
- Tx Generator를 `mn-node-toolkit`으로 대체했습니다.
- 새로운 Node Ledger Prometheus 메트릭을 추가했습니다.
- 트랜잭션 제너레이터에서 외부 증명 서버 지원을 추가했습니다.
- `testnet-02` 배포를 위한 `addresses.json` 및 체인 사양을 추가했습니다.
- 네이티브 토큰 관리 팔렛과 메인 체인 팔로워를 추가했습니다.
- 이미지에 더 많은 디버깅 도구를 기본 포함했습니다.
- 트랜잭션 제너레이터에 testnet 네트워크 지원을 추가했습니다.
- 새로운 `midnight_zswapStateRoot` RPC 메서드를 추가했습니다.

#### Changed

- 체인 스펙을 선언적으로 생성하도록 변경했습니다(더 DRY하고 오류 발생 가능성 감소).
- 하위 명령어의 CLI 오류를 개선했습니다.
- cNIGHT 관련 팔렛에 제네시스 값을 추가했습니다.
- 런타임 모델 Events의 열거형 충돌을 수정했습니다.
- `BASE_PATH` 디렉토리가 존재하지 않을 때 시작 시 충돌하는 문제를 수정했습니다.
- Partner-Chains의 from-env 호출을 자체 설정으로 대체했습니다.
- Partnerchains 의존성을 1.4.0으로 업데이트했습니다.
- 트랜잭션 제너레이터의 전송 메커니즘을 개선했습니다.
- 새로운 스토리지 모델로 전환했습니다.
- Partnerchains 의존성을 1.3.1로 업데이트했습니다.
- 이전 원장 버전을 제거했습니다(하위 호환성 없음).
- TTY 없는 분리 모드에서 작동하도록 partnerchains-dev 이미지를 수정했습니다.
- 노드 기본 이미지를 debian unstable에서 debian bookworm으로 전환했습니다.
- 트랜잭션 검증 및 처리를 수정했습니다.
- 빌드에 디버그 정보를 포함시켰습니다.
- 사용하지 않는 `CFG_PRESET=testnet-preview`를 제거했습니다.
- CI를 수동 디스패치로 변경했습니다.
- Token Management 크레이트를 분리했습니다.
- 필드/커브를 Pluto-Eris에서 BLS12-381로 전환했습니다.
