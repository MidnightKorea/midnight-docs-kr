## Midnight 0.3.6 RC1 release notes

기존 qa-net/devnet 환경(Plonk 기반 증명 시스템에만 기반한 원장이 적용된 환경)을 위한 Midnight 노드 릴리스입니다.

### New Features

- devnet 환경 테스트를 위해 트랜잭션 한도를 낮추었습니다.
- `seed` 인자를 선택 사항으로 변경했습니다.
- 트랜잭션 제너레이터를 Plonk 브랜치로 백포트했습니다.
- Plonk 브랜치에 스토리지 모니터를 추가했습니다.
- `d` 파라미터 및 위원회 모드를 오버라이드하는 기능을 도입했습니다.
- `d` 파라미터 오버라이드가 포함된 런타임 WASM을 생성했습니다.
- `qanet` 체인 스펙 및 `addresses.json`을 업데이트했습니다.

### Bug Fixes

- WSS RPC와의 호환성을 확보하기 위해 트랜잭션 제너레이터의 Docker 빌드를 수정했습니다.
- 부트 노드의 `seed` 파라미터 관련 문제를 해결했습니다.
- `cargo.lock` 파일을 업데이트했습니다.
- 모든 E2E 테스트를 수정하고 활성화했습니다.

### Breaking Changes

- **Node 0.3.6**으로 업데이트되어 이전 버전을 사용하는 환경에서는 호환성 조정이 필요합니다.
- 새로운 `d` 파라미터 오버라이드가 포함된 스펙 버전 `0.2.3`으로 업데이트했습니다.

### Known Issues

- 사용자 정의 환경에서 낮아진 트랜잭션 한도를 테스트할 때 사소한 불일치가 발생할 수 있습니다.
- 고부하 시나리오에서 스토리지 모니터에 대한 추가 최적화가 필요할 수 있습니다.

### Upgrade Instructions

1. 최신 변경 사항을 풀(pull)하여 노드를 **0.3.6** 버전으로 업데이트하세요.
2. 선택 사항이 된 `seed` 인자에 맞게 설정을 조정하세요.
3. devnet 환경에서 새로운 트랜잭션 한도를 테스트하여 호환성을 확인하세요.
4. 업데이트된 `d` 파라미터 및 위원회 모드 설정에 맞게 워크플로를 확인하세요.
5. 새로운 `d` 파라미터 오버라이드가 포함된 `0.2.3` 버전으로 체인 스펙을 업데이트하세요.
