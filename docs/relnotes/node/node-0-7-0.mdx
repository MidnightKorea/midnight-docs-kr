## Midnight 0.7.0 release notes

기능 향상, 성능 최적화, 개발자 경험 개선을 포함하는 릴리스입니다. RPC 오류 처리를 재작업하여 오류 보고가 명확해졌으며, 기존 인덱서와의 하위 호환성은 유지됩니다.

### Compatible Versions

이번 릴리스는 다음 컴포넌트와의 호환성이 테스트되었습니다:

- **Cardano**: 10.1.2  
- **PartnerChains**: 1.2  
- **PartnerChains Smart Contract**: 6.2.1  
- **DbSync**: 13.5.0.2  
- **Kupo**: 2.9.0  
- **Ogmios**: 6.5.0  
- **Midnight Ledger**: 3.0.2  
- **Midnight Lace Wallet**: 1.2.3  
- **Scala Indexer**: 2.2.7  
- **Midnight JS**: 0.25  

### New Features

- `pallet-tx-pause`를 통한 extrinsic 호출 일시정지/재개 거버넌스 기능 추가
- Runtime API에서 LedgerParameters와 TransactionCostModel 노출
- 업그레이드 투표 임계값 설정 기능 추가
- `BOOTNODES` 환경 변수 추가로 설정 간소화
- Docker 이미지에 **curl** 포함 (`docker-compose` 헬스 체크 지원)
- Docker 이미지에 `bytehound` 추가 (라이브 메모리 프로파일링)
- Earthly 타겟 `+generate-keys-qanet` 추가로 키 생성 간소화
- testnet-preview QA 환경용 설정 파일 추가

### Changes

- Rust 컴파일러 1.78 → 1.82 업그레이드
- ParityDB 사용에 맞춰 가중치 업데이트 (RocksDB 대체)
- 트랜잭션 제너레이터/업그레이더 도구의 비보안 RPC 엔드포인트 연결 지원
- Ledger API 리팩토링: `midnight_ledger`에서 `midnight_pallet` 및 Runtime API로 오류 전파
- Ledger API v2 활성화
- 노드 이미지 크기 약 1.7GB 축소
- 설정 프리셋에서 상대 경로 사용 (이식성 향상)
- 체인 스펙에 기존 부트노드 주소 포함 (수동 설정 축소)

### Bug Fixes

- 여러 `MidnightRuntimeApi` 버전을 처리하도록 RPC 엔드포인트를 수정했습니다.
- 트랜잭션 검증 시 메모리 소비 문제를 수정했습니다.
- mint coins 호스트 함수 관련 문제를 해결했습니다.
- QA 환경에서의 JSON 블록 RPC 메서드 문제를 수정했습니다.
- ARM 아키텍처에서의 트랜잭션 제너레이터 안정성을 개선했습니다.
- 체인 스펙 부트노드를 수정했습니다.

### Breaking Changes

- 런타임 변경 사항이 새로운 `Ledger API v2` 기능을 노출합니다.
- 재작업된 RPC 오류 처리로 인해 조정이 필요할 수 있습니다.
- 런타임 설정에서 세션 초기화 순서를 업데이트했습니다.

### Known Issues

- 라이브 메모리 프로파일링에 대한 추가 최적화가 계획되어 있습니다.

### Upgrade Instructions

1. 최신 변경 사항과 자산을 pull하여 노드를 **0.7.0**으로 업데이트하세요.
2. Rust 컴파일러를 **1.82**로 업데이트하세요.
3. 체인 스펙에 업데이트된 부트노드 주소를 반영하세요.
4. Ledger API v2 및 Runtime API 업데이트로 트랜잭션 워크플로를 테스트하세요.
5. 필요시 `pallet-tx-pause`로 거버넌스 설정을 구성하세요.
