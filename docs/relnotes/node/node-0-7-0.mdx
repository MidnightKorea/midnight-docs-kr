## Midnight 0.7.0 release notes

이번 릴리스는 기능 향상, 성능 최적화, 개발자 경험 개선을 위한 다양한 개선 사항과 새로운 기능을 도입합니다. 노드에는 보다 명확한 오류 보고를 위해 재작업된 RPC 오류 처리가 포함되어 있으며, 기존 인덱서와의 하위 호환성을 유지합니다.

### Compatible Versions

이번 릴리스는 다음 컴포넌트와의 호환성이 테스트되었습니다:

- **Cardano**: 10.1.2  
- **PartnerChains**: 1.2  
- **PartnerChains Smart Contract**: 6.2.1  
- **DbSync**: 13.5.0.2  
- **Kupo**: 2.9.0  
- **Ogmios**: 6.5.0  
- **Midnight Ledger**: 3.0.2  
- **Midnight Lace Wallet**: 1.2.3  
- **Scala Indexer**: 2.2.7  
- **Midnight JS**: 0.25  

### New Features

- `pallet-tx-pause`를 통해 extrinsic 호출을 일시정지/재개하는 거버넌스 기능을 추가했습니다.
- 새로운 Runtime API를 통해 LedgerParameters와 TransactionCostModel을 노출했습니다.
- 업그레이드에 필요한 투표 임계값을 설정하는 기능을 도입했습니다.
- 보다 쉬운 설정을 위해 `BOOTNODES` 환경 변수를 추가했습니다.
- `docker-compose`를 통한 헬스 체크를 지원하기 위해 Docker 이미지에 **curl**을 포함시켰습니다.
- Docker 이미지에 `bytehound`를 추가하여 라이브 메모리 프로파일링을 활성화했습니다.
- 보다 쉬운 키 생성을 위해 새로운 Earthly 타겟 `+generate-keys-qanet`을 도입했습니다.
- 새로운 testnet-preview QA 환경을 위한 설정 파일을 추가했습니다.

### Changes

- Rust 컴파일러를 1.78에서 1.82로 업그레이드했습니다.
- RocksDB 대신 ParityDB를 사용하도록 가중치를 업데이트했습니다.
- 트랜잭션 제너레이터와 업그레이더 도구가 비보안 RPC 엔드포인트에 연결할 수 있도록 했습니다.
- Ledger API를 리팩토링하여 `midnight_ledger`에서 `midnight_pallet` 및 Runtime API로 오류를 전파하도록 했습니다.
- 새로운 Ledger API v2를 활성화했습니다.
- 노드 이미지 크기를 약 1.7GB 줄였습니다.
- 이식성 향상을 위해 설정 프리셋에서 상대 경로를 사용하도록 변경했습니다.
- 수동 설정을 줄이기 위해 기존 부트노드 주소가 포함된 체인 스펙을 업데이트했습니다.

### Bug Fixes

- 여러 `MidnightRuntimeApi` 버전을 처리하도록 RPC 엔드포인트를 수정했습니다.
- 트랜잭션 검증 시 메모리 소비 문제를 수정했습니다.
- mint coins 호스트 함수 관련 문제를 해결했습니다.
- QA 환경에서의 JSON 블록 RPC 메서드 문제를 수정했습니다.
- ARM 아키텍처에서의 트랜잭션 제너레이터 안정성을 개선했습니다.
- 체인 스펙 부트노드를 수정했습니다.

### Breaking Changes

- 런타임 변경 사항이 새로운 `Ledger API v2` 기능을 노출합니다.
- 재작업된 RPC 오류 처리로 인해 조정이 필요할 수 있습니다.
- 런타임 설정에서 세션 초기화 순서를 업데이트했습니다.

### Known Issues

- 라이브 메모리 프로파일링에 대한 추가 최적화가 계획되어 있습니다.

### Upgrade Instructions

1. 최신 변경 사항과 자산을 풀(pull)하여 노드를 **0.7.0** 버전으로 업데이트하세요.
2. 호환성을 보장하기 위해 Rust 컴파일러를 **1.82** 버전으로 업데이트하세요.
3. 업데이트된 부트노드 주소를 체인 스펙에 통합하세요.
4. 새로운 Ledger API v2 및 Runtime API 업데이트로 트랜잭션 워크플로를 테스트하고 검증하세요.
5. 필요한 경우 extrinsic 일시정지/재개를 위한 새로운 `pallet-tx-pause`를 사용하여 거버넌스 설정을 구성하세요.
