## Midnight 3.0.6 Proof Server release notes

새로운 기능, 버그 수정, 호환성에 영향을 주는 변경 사항 등 Midnight Proof Server의 최신 업데이트와 개선 내용을 안내합니다.

## New Features

- `coinCommitment`과 `coinNullifier` 함수를 도입했습니다.
- Zswap 패키지에 키를 Zswap 로컬 상태에서 분리하는 API을 도입했습니다.
- `midnight-storage-0.3`에서 `Map`의 직렬화 형태가 변경되는 호환성 문제를 처리했습니다. 이로 인해 중첩된 `Map` 구조를 가진 모든 타입에 마이너 버전 업데이트가 필요했습니다.

## Breaking Changes

- 추가된 private 필드로 인해 일부 데이터 구조를 직접 생성할 수 없게 되었습니다:
  - `Transaction`은 더 이상 공개 구조 enum이 아닙니다. 컴포넌트가 `StandardTransaction`과 `ClaimMintTransaction`으로 분리되었으며, 이들은 이제 private입니다.
  - `ContractCall`, `ContractDeploy`, `MaintenanceUpdate`이 이제 private입니다.
  - 트랜잭션을 생성하려면 `transaction-construction` 기능과 해당 메서드를 사용하세요.
- `midnight-storage-0.3`의 변경으로 직렬화가 영향을 받아 중첩된 `Map` 구조를 가진 타입의 업데이트가 필요합니다.

## Bug Fixes

- 인메모리 환경에서 잘못된 정보를 사용하는 Schnorr 증명의 보안 버그를 수정했습니다.
- 원장에서 발생 가능한 다양한 panic 인스턴스를 제거했습니다.
- 잔액 확인의 취약점을 제거했습니다(소수의 트랜잭션에 호환성 영향).
- 직렬화 후 Schnorr 증명이 유지되지 않는 문제를 수정했습니다.
- 인메모리 환경에서의 Schnorr 증명 보안 관련 이슈를 수정했습니다.
- 트랜잭션 재직렬화 이슈를 수정했습니다.
- WASM `checkProofData` 엔드포인트를 수정했습니다.
- WASM API `fees` 엔드포인트를 수정했습니다.
- 유지보수 업데이트의 Schnorr 증명이 직렬화 후 유지되지 않는 문제를 수정했습니다.
- proof server에서 디버그 로깅 제거를 수정했습니다.
